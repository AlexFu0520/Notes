*Algorithm :* [[巴什博弈]]
## 描述
对于给定的 $n$  个石子，两位玩家轮流行动，按以下规则取石子：  
- 每人每次可以取走 $x(l≦x≦r)$ 个石子，即如果任意时刻，剩余石子数量小于  $l$  ，则不能再取；  
- 拿到最后一个石子的一方获胜。  
如果双方均采用最优策略，判断先手能否必胜。
### 输入描述：
每个测试文件均包含多组测试数据。第一行输入一个整数  $T(1≦T≦2×10^6)$  代表数据组数，每组测试数据描述如下：  

在一行上输入三个整数  $n,l,r(1≦n,l,r≦109;l≦r)$  代表石子数量、单次取走的下限、单次取走的上限。
### 输出描述：
对于每一组测试数据，如果先手必胜，在一行上输出 $YES$ ；否则，直接输出 $NO$。
## 示例1

输入：
```plaintext
3
3 5 7
9 2 4
7 2 5
```

输出：

```plaintext
NO
YES
NO
```
### 思路：
本题是巴什博弈的扩展形式。标准的巴什博弈允许取  $[1~,~m]$ 个石子，而本题允许取  $[l~,~r]$  个。解题的核心思想仍然是寻找游戏状态的周期性，并划分出周期内的“必胜态”与“必败态”。

1. **寻找状态周期** 通过博弈论的归纳分析可以证明，此类取石子游戏的胜负状态是以 $r + l$ 为周期循环的。也就是说，一个有  $n$ 个石子的局面，其胜负性质等同于一个有 $n~(mod~l+r)$ 个石子的局面。
2. **划分周期内的必败态** 在一个周期内，即石子数  $n$ 满足 $0\leq n\leq l+r$  时，我们需要确定哪些是必败态。
    - 根据规则，如果轮到某个玩家时，石子数小于 $l$ ，该玩家就无法行动，直接判负。
    - 因此，石子数在区间 $[0,l-1]$  内的局面都是**必败态**。
3. **划分周期内的必胜态**
    - 如果一个局面 $n$ 允许玩家通过一次操作（取 $k$ 个石子）将局面转移到一个必败态 $n-k$ ，那么局面 $n$ 就是一个**必胜态**。
    - 对于区间 $[l,l+r-1]$ 内的任何局面 ，玩家总可以找到一个合法的取石子数量 $k$ (其中 $l\leq k\leq r$ ) ，使得剩余的石子数 $n-k$ 落在必败区间 $[0,l-1]$  中。因此，这些局面都是**必胜态**。
4. **最终结论** 结合以上两点，我们可以得到一个简单的判断法则：
    - 首先计算 $n$ 在一个周期内的等效状态：$r = n~(mod~l+r)$。
    - 然后判断这个等效状态 是否落在必败区间内。
    - 如果 $r<l$ ，则初始局面  是一个必败态，先手必败（**Bob 胜**）。
    - 如果 $r\geq l$ ，则初始局面  是一个必胜态，先手必胜（**Alice 胜**）。
### code:
```cpp
#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

void solve() {
    int n,l,r;
    cin >> n >> l >> r;

    cout << (n % (r + l) >= l ? "YES" : "NO") << '\n';
}               

int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    cout.tie(nullptr);


    int T = 1;
    cin >> T;
    while (T--) {
        solve();   
    }
    return 0;
}
```
