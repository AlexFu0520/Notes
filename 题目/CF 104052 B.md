<center>time limit per test: 2 seconds</center>
<center>memory limit per test: 512 megabytes</center>
<center>input: standard input</center>
<center>output: standard output</center>

*Algorithm：*[[数学算法]]

The school canteen has four types of fruits: apricots, bananas, apples, and pears. There are three different possible sets of fruits available for lunch:

1.  two apricots, one banana, and one apple;
2.  two apricots and two apples;
3.  one apricot, one banana, two apples, and one pear.

The canteen's employees want to assemble as many lunch fruit sets for children as possible. Help them do it!

Since the number of fruits delivered is different every day, you need to solve the problem for multiple test cases.
### Input
The first line of the input contains an integer $t$ ($1 \le t \le 10^4$) — the number of test cases.

The following $t$ lines contain a description of each test case. Each line contains four integers $a$, $b$, $c$, and $d$ ($1 \le a, b, c, d \le 10^9$) — the number of apricots, bananas, apples, and pears, respectively.
### Output
For each test case, print the maximum number of fruit sets it's possible to make.
### Scoring

| **Subtask** | **Score** | **Constraints**                   |
| ----------- | --------- | --------------------------------- |
| $1$         | $10$      | $t \le 10$; $a, b, c, d \le 10$   |
| $2$         | $20$      | $t \le 10^4$; $a, b, c, d \le 10$ |
| $3$         | $20$      | $t \le 10$; $a, b, c, d \le 200$  |
| $4$         | $20$      | $t \le 10$; $a, b, c, d \le 10^6$ |
| $5$         | $30$      | No additional constraints         |
### Example

**Input**
```plaintext
6
3 3 3 3
3 1 4 1
4 3 2 1
3 3 6 5
9 7 6 7
9 10 10 6
```

**Output**
```plaintext
2
2
2
3
5
6
```

**Note**
In the first case, you can make two sets of fruits: one of type $1$ and one of type $3$.
In the second case, one of the possible answers is $(2, 3)$.
In the third case, the optimal answer is $(1, 1)$.
In the fourth case, you can make three sets of type $3$.

### 思路：
1. 枚举所有水果子集 T
2. 计算 T 的总供给 cur
3. 计算单份套餐对 T 的最小消耗 to_check
4. 计算 T 允许的最大 k ：cur / to_check
5. 计算所有 T 的 k 上限取最小值，即为答案
### code:
```cpp
#include <iostream>
#include <vector>
#include <algorithm>
typedef long long ll;
using namespace std;

//套餐
vector<vector<int>> tmp = {
    {2, 1, 1, 0},
    {2, 0, 2, 0},
    {1, 1, 2, 1}
};
void solve() {
	// nums 负责存储4种水果的数量
    vector<int> nums(4);
	for (auto &x: nums) cin >> x;

	//枚举4种水果的所有子集（2^4 = 16），用4位二进制表示
	//这一步对应霍尔定理中任意子集的要求
	ll ans = inf;
	for (int i = 0; i < 16; i ++) {
		ll cur = 0;
		// cur表示子集中所有水果的总数量
		//这代表的是霍尔定理的供给侧
		for (int j = 0; j < 4; j ++) {
			//j 表示的是四种水果
			// 用二进制表示第i个子集是否有水果j
			if (i >> j & 1)  {
				cur += nums[j];
			}
		}
		
		//计算子集的单份最小消耗（霍尔定理的需求侧）
		int to_check = inf;
		for (int j = 0; j < 3; j ++) { //遍历三种套餐
			int val = 0;
			for (int k = 0; k < 4; k ++) {
				if (i >> k & 1) { //累加该套餐对子集的消耗
					val += tmp[j][k];
				}
			}
			to_check = min(to_check, val); //取三种套餐的最小消耗
		}
		//霍尔定理的需求侧逻辑为：要做 k 份套餐，不管选择哪种套餐，每份都至少消耗to_check数量的子集T，因此总需求为 k * to_check
		if (to_check) ans = min(ans, cur / to_check);
	}

	cout << ans << '\n';
}               

int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    cout.tie(nullptr);


    int T = 1;
    cin >> T;
    while (T--) {
        solve();   
    }
    return 0;
}
```
### 逐个测试用例拆解

#### 1. 测试用例 1：输入 `3 3 3 3` → 输出 `2`
枚举关键子集（决定最终最小值的子集）：
- **子集 T = 杏 + 苹果**（二进制`0101`，选中杏和苹果）：
    - `cur` = 杏数量 + 苹果数量 = 3 + 3 = 6；
    - `to_check`：计算 3 种套餐对该子集的消耗，取最小值：
        - 套餐 1：2（杏）+1（苹果）=3；
        - 套餐 2：2（杏）+2（苹果）=4；
        - 套餐 3：1（杏）+2（苹果）=3；
        - `to_check = min(3,4,3) = 3`； 
    - 该子集的 k 上限 = 6 / 3 = 2；
- 其他子集的 k 上限均≥2（比如杏子集的 k 上限 = 3/1=3），因此最终答案为 2。
#### 2. 测试用例 2：输入 `3 1 4 1` → 输出 `2`
关键子集：
- **子集 T = 杏 + 香蕉**（二进制`0011`，选中杏和香蕉）：
    - `cur` = 3 + 1 = 4；
    - `to_check`：    
        - 套餐 1：2+1=3；套餐 2：2+0=2；套餐 3：1+1=2；
        - `to_check = 2`；    
    - k 上限 = 4 / 2 = 2；
- 其他子集的 k 上限≥2（比如苹果子集 k 上限 = 4/1=4），答案为 2。
#### 3. 测试用例 3：输入 `4 3 2 1` → 输出 `2`
关键子集：
- **子集 T = 苹果**（二进制`0100`，仅选中苹果）：
    - `cur` = 2；
    - `to_check`：套餐 1 耗 1、套餐 2 耗 2、套餐 3 耗 2 → `to_check=1`；
    - k 上限 = 2 / 1 = 2；
- 其他子集的 k 上限≥2（比如杏子集 k 上限 = 4/1=4），答案为 2。
#### 4. 测试用例 4：输入 `3 3 6 5` → 输出 `3`
关键子集：
- **子集 T = 杏**（二进制`0001`，仅选中杏）：
    - `cur` = 3；
    - `to_check`：套餐 1 耗 2、套餐 2 耗 2、套餐 3 耗 1 → `to_check=1`；
    - k 上限 = 3 / 1 = 3；
- 其他子集的 k 上限≥3（比如苹果子集 k 上限 = 6/1=6），答案为 3。
#### 5. 测试用例 5：输入 `9 7 6 7` → 输出 `5`
关键子集：
- **子集 T = 杏 + 苹果**（二进制`0101`）：
    - `cur` = 9 + 6 = 15；
    - `to_check`：套餐 1 耗 3、套餐 2 耗 4、套餐 3 耗 3 → `to_check=3`；
    - k 上限 = 15 / 3 = 5；
- 其他子集的 k 上限≥5（比如香蕉子集 k 上限 = 7/1=7），答案为 5。
#### 6. 测试用例 6：输入 `9 10 10 6` → 输出 `6`
关键子集：
- **子集 T = 杏 + 苹果**（二进制`0101`）：  
    - `cur` = 9 + 10 = 19；
    - `to_check`：套餐 1 耗 3、套餐 2 耗 4、套餐 3 耗 3 → `to_check=3`；
    - k 上限 = 19 / 3 = 6（整数除法，向下取整）；  
- 其他子集的 k 上限≥6（比如梨子集跳过，香蕉子集 k 上限 = 10/1=10），答案为 6。
