描述
--

对于给定的由 $n$ 个整数组成的数组 $\{a_1,a_2,\dots,a_n\}$，小龙和小蛇借助于此数组进行游戏。  
游戏步骤如下：  
${\hspace{20pt}}_\texttt{1.}\,$小龙选择一个非空区间 $[a, b]$；  
${\hspace{20pt}}_\texttt{2.}\,$小蛇选择一个非空区间 $[c, d]$；  
${\hspace{20pt}}_\texttt{3.}\,$将选中的区间中的全部元素均乘上 $k$，得到数组 $a'$；  
游戏只进行一轮，三个步骤结束后立即停止。  
小龙想要让数组 $a'$ 的元素之和尽可能大，小蛇想要让数组 $a'$ 的元素之和尽可能小。假设双方都采取的是最优策略，请你计算操作后得到的数组 $a'$ 的元素之和。  
  
请注意，区间 $[a, b]$ 和 $[c, d]$ 可以相交，但只结算一次，即若某一个位置被小龙和小蛇同时选中，依旧只乘一次。

### 输入描述：

每个测试文件均包含多组测试数据。第一行输入一个整数 $T\left(1\leqq T\leqq 100\right)$ 代表数据组数，每组测试数据描述如下：  
第一行输入两个整数 $n,k\left(1\leqq n\leqq 10^5;\ -100\leqq k\leqq 100\right)$ 代表数组中的元素数量、乘数。  
第二行输入 $n$ 个整数 $a_1,a_2,\dots,a_n\left(-10^6\leqq a_i\leqq 10^6\right)$ 代表数组元素。  
  
除此之外，保证单个测试文件的 $n$ 之和不超过 $2 \times 10^5$。

### 输出描述：

对于每一组测试数据，新起一行。输出一个整数，代表操作后数组 $a'$ 的元素之和。

示例1
---

输入：
```txt
3
2 4
1 1
6 0
1 1 4 5 1 4
4 -1
-2 1 -10 3
```

输出：
```txt
8
0
8
```

说明：对于第一组测试数据，小龙的最优策略是选择区间 $[1,2]$，一旦这么做了，无论小蛇选择的区间是什么，都不会影响最终答案。

对于第三组测试数据，其中一种最优策略为：龙选择区间 $[1,3]$，小蛇选择区间 $[4,4]$。
## 数学模型：
1. 给定数组a的元素总和位 $S = \sum_{i=1}^na_{i}$ ，设小龙选择的区间为 $I_{L}$ ，小蛇选择的区间为 $I_{S}$ ，区间并集（即被选中的元素集合）为 $U=I_{L}\cup I_{S}$ 。
2. 选中元素的总和为 $V=\sum_{i\in U}a_{i}$ 。执行操作后，数组的总和为 $S'$ 可表示为：$$S'=\sum_{i\notin U}a_{i} +\sum_{i\in U}(k*a_{i})=S+(k-1)V$$
3. 令 $D=k-1$ ，小龙的目标是最大化 $S'$ ，小蛇的目标是最小化 $S'$ 
	- 若$D >0$（即 $k > 1$）：小龙希望最大化$V$,小蛇希望最小化$V$
	- 若$D<0$（即 $k<1$）：小龙希望最小化$V$,小蛇希望最大化$V$
	- 若$D=0$（即 $k=1$）：$S'=S$ ，此时博弈策略不影响结果
## 博弈策略：
1. 小蛇的策略分析（针对$D<0$的情形）：此时小龙希望最小化$V$，小蛇希望最大化$V$，由于$I_{S}$可以是数组中的任意非空区间，小蛇总可以执行以下操作：
	- 无论小龙选择的$I_{L}$是什么，小蛇都可以选择$I_{S}=[1,n]$（整个数组）
	- 一旦小蛇选择$I_{S}=[1,n]$,由于$I_{L}\cup I_{S}=[1,n]$ ,则$V$必然等于$S$（整个数组的和）
	- 这意味着，在$D<0$是，小蛇保证了$V$至少能够达到$S$（即小蛇可以强行让$V\geq S$）
	- 同理，由于小龙可以第一步先手就选择 $I_{L}=[1,n]$，此时无论小蛇怎么选，$V$都只能是$S$

## 思路：
1. 根据博弈逻辑推导：在双方均采取最优策略的情况下，操作后的数组元素之和始终等于元数组元素总和乘以系数k
2. 简单理解，对于原数组中的任意一个元素，肯定对于小蛇、小龙两人，有如下两种情况：  
	- 其中一方有明显效益（元素值非0）：会被该方选中，然后考虑到重叠部分只算一次，选择后，另一方也可以选择该元素（反正已经无负面影响）；  
	- 都无明显效益（元素值为0）：两方都选上，然后可以继续往后选择，扩大自己的选择区间\[a,b]，谋求更大效益。  
  
最后双方都可以选择整个原数组，所以结果就是原数组和乘上k。

#include <iostream>
#include <vector>
#include <algorithm>
#include <queue>
using namespace std;
typedef long long ll;

void solve() {
    int n,k;
    ll s = 0;
    cin >> n >> k;
    vector<int> a(n);
    for (int i = 0 ; i < n ; ++i) {
        cin >> a[i];
        s += a[i];
    }
    
    cout << s * k << '\n';
}

int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    cout.tie(nullptr);

    int T = 1;
    cin >> T;
    while (T--) {
        solve();   
    }
    return 0;
}
