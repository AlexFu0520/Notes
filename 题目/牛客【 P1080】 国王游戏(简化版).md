*Algorithm :* [[贪心算法]]，[[二维数组sort]]
## 描述

龙国国庆之际，国王邀请  $n$ 位大臣参与一起玩一个游戏。国王与每位大臣都会在左、右手各写一整数：国王分别写下了 $(a_{0},b_{0})$ ；此后第 ii 位大臣写下 $(a_{i}​,b_{i}​)$ 。  
  
现在国王站在最前方，其余大臣依次排成一行。排定顺序后，第 ii 位大臣（排在队伍中的相对顺序）能得到的金币数为  $\left \lfloor \frac{{\textstyle \prod_{j=0}^{i-1}a_{j}} }{b_{i}} \right \rfloor$   ，其中 $a_{0}=A$ 表示国王左手的数字。  

国王希望通过调整大臣的站位顺序（相对顺序可任意改变，国王位置固定在最前），使得 **获得金币最多** 的大臣得到的金币数尽可能小。  
### 输入描述：
第一行输入整数 $n(1≦n≦60)$ ——大臣人数。  
第二行输入两个整数  $a_{0},b_{0}(1≦A,B≦8)$ ——国王左、右手的数字。  
随后 $n$ 行，第 $i$ 行输入两个整数 $a_{i},b_{i}(1≦ai,bi≦8)$ ——第 $i$ 位大臣左、右手的数字。  
### 输出描述：
输出一个整数，表示在最优站位下，获奖最多的大臣所能获得的最小金币数。
数据保证答案不超过 10181018；  
## 示例1
输入：
```plaintext
3
1 1
2 3
7 4
4 6
```
输出：
```plaintext
2
```
说明：
按 11、22、33 这样排列队伍，获得奖赏最多的大臣所获得金币数为 22；  
按 11、33、22 这样排列队伍，获得奖赏最多的大臣所获得金币数为 22；  
按 22、11、33 这样排列队伍，获得奖赏最多的大臣所获得金币数为 22；  
按22、33、11这样排列队伍，获得奖赏最多的大臣所获得金币数为 99；  
按 33、11、22这样排列队伍，获得奖赏最多的大臣所获得金币数为 22；   
按33、22、11 这样排列队伍，获得奖赏最多的大臣所获得金币数为 99。  
因此，奖赏最多的大臣最少获得 22 个金币，答案输出 22。
### 思路：
1. 这是一个典型的 minimax(最小化最大值) 问题，通常可以利用贪心算法找到最优解，这问题的关键就在于确定大臣的最佳排序策略
2. 我们可以通过 “微扰 / 邻项交换” 的思想来推导出排序的规则
### 数学证明：
设在当前队列中，有两个相邻的大臣为 i 和 j  $(i = j - 1)$ ，我们来分析交换他们的位置对结果的影响
设国王与前面的大臣的累乘积为 $S (S = {\textstyle \prod_{k=0}^{i-1}a_{k}})$ 
1. 顺序为（i，j）时
	- 大臣 i 的奖励为 $C_{i} = \lfloor\frac {S}{b_{i}}\rfloor$ 
	- 大臣 j 的奖励为 $C_{j}=\lfloor \frac {S~\cdot~a_{i} } {b_{j}} \rfloor$ 
	- 此时，这两位大臣以及他们之后所有大臣中，奖励的最大值主要由  $C_{j}$ 决定（因为分母不变，分子在累积）。所以我们关注这两个位置产生的最大奖励，即 $max(C_{i}~,~C_{j})$ 。
2. 顺序为（j ，i）时
	- 大臣 i 的奖励为 $C_{i}^{'} = \lfloor\frac {S~\cdot~a_{i}}{b_{i}}\rfloor$ 
	- 大臣 j 的奖励为 $C_{j}^{'}=\lfloor \frac {S} {b_{j}} \rfloor$ 
	- 这两个位置产生的最大奖励为 $max(C_{i}^{'}~,~C_{j}^{'})$ 
要使全局的最大奖励最小，我们必须保证任意两项的安排都是最优的
我们希望顺序（i，j）比顺序（j，i）更优，这意味着（i，j）产生的局部最大奖励应该小于等于（j，i）产生的局部最大奖励 :
$$max(C_{i}~,~C_{j}) \leq max(C_{i}^{'}~,~C_{j}^{'})$$
忽略取整符号 (它不影响大小关系) 和 累乘积 $S$ ,可得：
$$max(\frac{1}{b_{i}}~,~\frac{a_{i}}{b_{j}}) \leq max(\frac{a_{j}}{b_{i}}~,~\frac{1}{b_{j}})$$
为了方便比较，两边同时乘以 $b_{i} \cdot b_{j}~~(b_{i}\cdot b_{j}>0)$ 
$$max(b_{j}~,~a_{i}\cdot b_{i}) \leq max(a_{j}\cdot b_{j}~,~b_{i})$$
之后我们来证明，如果 $a_{i}\cdot b_{i}\leq a_{j}\cdot b_{j}$ ，上述恒等式恒成立

### 证明：
1. 假设 $a_{i}\cdot b_{i}\leq a_{j}\cdot b_{j}$ ，我们需要证明 $max(b_{j}~,~a_{i}\cdot b_{i}) \leq  max(a_{j}\cdot b_{j}~,~b_{i})$ 
2. 我们使用反证法，假设该不等式不成立，即 $max(b_{j}~,~a_{i}\cdot b_{i}) > max(a_{j}\cdot b_{j}~,~b_{i})$ 
3. 因为 $a_{i}\cdot b_{i}\leq a_{j}\cdot b_{j}$ ，所以要使左边大于右边，必须有 $b_{j} >max(b_{i}~,~a_{j}\cdot b_{j})$ 
4. 这导出两个不等式：
$$b_{j}>b_{i}$$
$$b_{j}>a_{j}\cdot b_{j}$$
5. 从 $b_{j}>a_{j}\cdot b_{j}$ 中可以推出 $1 > a_{j}$ ，这与题目所给的条件相矛盾，所以此假设不成立，原不等式恒成立
### 结论：
当 $a_{i}\cdot b_{i}\leq a_{j}\cdot b_{j}$ 时，将大臣 i 排在 j 前面会得到不劣于交换后的解，因此我们的贪心策略为将所有大臣按照 $a_{i}\cdot b_{i}$ 的值从小到大排序

### code:
```cpp
#include <iostream>
#include <vector>
#include <algorithm>
typedef long long ll;
using namespace std;

void solve() {
    ll n,a0,b0;
    cin >> n;
    cin >> a0 >> b0;
    vector<vector<ll>> nums(n);
    for (ll i = 0 ; i < n ; ++i) {
        ll a,b;
        cin >> a >> b;
        nums[i] = {a,b};
    }
    // 对大臣进行排序
    sort(nums.begin(),nums.end(),[](const vector<ll>& num1,const vector<ll>& num2) {
        return num1[0] * num1[1] < num2[0] * num2[1];
    });
	
	//取出最大值
    ll sum = a0;
    ll mx = a0 / nums[0][1];
    for (int i = 1 ; i < n ; ++i) {
        sum *= nums[i - 1][0];
        mx = (sum / nums[i][1] > mx ? sum / nums[i][1] : mx);
    }
    cout << mx << '\n';
}               

int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    cout.tie(nullptr);


    int T = 1;
    // cin >> T;
    while (T--) {
        solve();   
    }
    return 0;
}
```
