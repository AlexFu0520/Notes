## 题目描述

给定一个正整数 $n$ 和 $n$ 个二元组 $(a_i,b_i)$，表示现在有 $b_i$ 个大小为 $a_i$ 的数。

定义一个可重集合的 $\operatorname{mex}$ 为最小的没有在这个集合中出现的自然数。

你需要将这 $\sum_{i=1}^{n} b_i$ 个数划分成 $k(k\ge 1)$ 个可重集合，使得这 $k$ 个可重集合的 $\operatorname{mex}$ 之和**恰好**为 $m$，并最小化这个 $k$。

现在有 $q$ 组询问，对于每一组询问给定一个 $m$，你需要输出最小的 $k$，如果无解则输出 $-1$。

## 输入格式

**本题包含多组测试数据。**

输入的第一行包含一个整数 $T$，表示测试数据的组数。

接下来包含 $T$ 组数据，每组数据的格式如下：

- 第一行包含两个整数 $n,q$，表示有 $n$ 个二元组并且存在 $q$ 次询问。

- 对于接下来的 $n$ 行每行输入两个整数，表示二元组 $(a_i,b_i)$。

- 对于接下来的 $q$ 行每行输入一个整数 $m$ 表示一次询问。

## 输出格式

对于每组测试数据输出 $q$ 行，每行包含一个整数，表示对应的答案。

## 说明/提示

**【样例 1 解释】**

对于 $m=0$ 和 $m=8$ 都可以证明不存在划分方案使得有解。

对于 $m=3$，可以将所有数划分为一个集合 $S=\{0,0,0,1,1,1,1,2,4\}$，这个集合的 $\operatorname{mex}$ 为 $3$。

对于 $m=4$，可以将所有数划分为两个集合 $S_1=\{0,0,1,1,1,1,2\}$ 和 $S_2=\{0,4\}$，这两个集合的 $\operatorname{mex}$ 之和为 $3+1=4$。

对于 $m=7$，可以将所有数划分为三个集合 $S_1=\{0,1,2,4\},S_2=\{0,1\},S_3=\{0,1,1\}$，这三个集合的 $\operatorname{mex}$ 之和为 $3+2+2=7$。

## 提示：
>**Hint 1**：哪些 $(a_{i},b_{i})$ 是有用的？

>**Hint 2**：如何贪心判断无解？

>**Hint 3**：如何用贪心求解答案？

>**Hint 4**：如何使用二分优化求解过程？
## 思路：
### Hint 1 解法：
1. 有一个比较容易发现的结论：找到最小的正整数 $i$ 满足 $a_{i} + 1\neq a_{i + 1}$ （若不存在则令 $i=n$），则我们只需要考虑 $(a_{1},b_{1}​)∼(a_{i}​,b_{i})$ ，剩下的 $(a_{i}+1​,b_{i}+1​)∼(a_{n}​,b_{n})$ 存不存在对答案无影响。
### Hint 2 解法：
1. 无解有两种情况：
	- $m=0$，若此时 $n \neq 0$ ，那么至少有一个集合内包含 $0$ ，那么 $mex$ 的总和一定不为 $0$，无解。
	- 假设目前的数能凑出来的最大 $mex$ 总和为 $x$，若 $m>x$ ，也无解。
2. 于是我们思考如何凑**最大**总和。有以下两点：
	- 在一个集合里每个数不会出现超过一次，不然就是浪费，因为没有对答案产生贡献。例如，$S=\left \{0,1,2,3\right \},S′= \left \{0,1,2,3,3\right \}$ ，两个集合的 $mex$ 都相等，但是后面一个集合多出来的 3 就浪费了。
	- 应当首先选 $mex$ 最大的集合来用。例如，假设 $z<w$ ，你现在有 $0∼w−1$ 各一个，如果你选择 $0∼z−1$ 组成一个 $mex=z$ 的集合，那么剩下的 $z∼w−1$ 这些数就浪费了，因此最优的应当选择 $0∼w−1$ 组成 $mex=w$ 的集合。

给出这两点，我们的策略就比较明显了，重复以下步骤即可：找到最大的 y 使得当前 $0∼y−1$ 都至少还有一个，然后将 $\left\{0,1,…,y−1\right \}$ 拿出来组成一个集合。

我们先考虑这个部分如何进行代码实现。对于 $i∈[1,n]$，记 $f_{i}=min_{j = 1}^i b_{j}$ ​，同时令 $g_{i}$​ 表示最小的满足 $b_{j}​=f_{i}$​ 的那个 $j$。然后对于上面那个求最大值的步骤，我们用如下实现方式：
1. 初始时记 $p=n,d=0$，$p$ 表示上面步骤中的那个 $y$，$d$ 表示当前删掉的 $0$ 的个数（这是因为，根据我们上面讨论的结果，$p$ 一定是不断越来越小的，那么如果当前某个下标 $i$ 所对应的 $i−1$ 这个数还能对答案产生贡献，则其一定和 $0$ 被删掉的个数是一样的）。
    
2. 此时 $c=f_{p}−d$ 就表示我们最多可以取出多少个 $\left\{0,1,…,p−1\right \}$。将它们全部取出来，贡献为 $p×(f_{p}​−d)$。
    
3. 令 $d←d+c$，表示我们又删掉了 $c$ 个 $0$ 。
    
4. 此时 $g_{p}−1$ 这个数已经被用完了（如果不明白请参考前面的定义），因此 $g_{p}​−1∼p−1$ 之间所有剩下的数全部失效，所以令 $p←g_{p}−1$。
    
5. 若 $p=0$，结束该过程。否则，回到步骤 2，重复该过程。
> **如何计算$f_{i}$ 和 $g_{i}$** 
> 考虑递推，计算 $f_{i}$​ 的时候，如果 $b_{i}​<f_{i}−1$​，那么更新 $f_{i}​=b_{i}​,g_{i}=i$；否则不变，即 $f_{i}​=f_{i}−1​,g_{i}​=g_{i}−1$​。正确性显然。

### Hint 3 解法：

我们考虑将这些组成最大值的最优集合全部存下来，可以使用一个数组 $e$ 来表示，$ei$​ 表示组成最大值的第 $i$ 个集合为 $\left\{0,1,…,ei​−1\right\}$ 。为了更快速地凑到 $m$，每次应当选择 $mex$ 最大的那个集合。查看前面我们的策略，发现 $e$ 恰好就是**单调不增**的！我们可以找到一个 $i$ 满足 $\sum_{j=1}^i e_{j} \leq m,\sum_{j = 1}^{i+1}e_{j} >m$ ，那么我们为了凑出 $m$，可以将第 $i+1$ 个集合拆开，只剩下 ${0,1,…,m−\sum_{j =1}^i​e_{j}}$（当然如果 $m=\sum_{j=1}^ie_{j}$ ​，可以直接不要这个集合），这样前 $i+1$ 个集合的 $mex$ 值加起来恰好等于 $m$！

那么多出来的东西怎么办？我们沿用之前的这条性质：

> 在一个集合里每个数不会出现超过一次，不然就是浪费，**因为没有对答案产生贡献**。

既然重复的数不会产生贡献，剩下的数我们全部扔到前面的集合里去就可以了！实际上可以直接全部扔到第 $1$ 个里，因为 $e_{1}=max~e_{i}$。

所以此时答案就是 $i+\left[ m=\sum_{j=1}^i e_{j}\right]$。
> **答案中的括号是什么意思**
> 
> 是艾弗森括号，当 p 条件成立时，$[p]=1$；否则 $[p]=0$。
### Hint 4 做法

**正解**

上一个做法的问题主要是对于很多很多完全相同的 $e_{i}$​，我们重复存了很多遍；于是不难想到改成维护两个数组 $e,f$，表示有 $f_{i}$​ 个 $e_{i}$​，此时就可以满足 $e_{i}​\neq e_{i}+1$​，则空间复杂度 $O(n)$。

时间方面，不难发现上面这个东西实则做完前缀和之后是二分查找板子，二分一下即可，复杂度可降为 $O(qlogn)$。

如何实现？记录 $h_{i}$ 表示 $\sum_{j=1}^i​e_{j}\times f_{j}$，首先在 $h_{i}$​ 中二分，找到那个满足 $h_{i}​≤m,h_{i}+1​>m$ 的 $i$，然后加上在第 $i+1$个整块中的数量，因此答案为：$$(\sum_{j=1}^if_{j})+\left \lceil \frac{m-h_{i}}{e_{i+1}} \right \rceil $$

时间复杂度 $O(qlogn)$，可以通过！

有一个小 $corner case$（当然大样例第一组就展示了），就是如果 $m<e_{1}$​，按照上面的做法，答案应当是 1，但是此时后面剩下的数**不能**全部塞进这唯一的一个集合里（证明留给读者），需要额外开一个集合，因此答案是 2。

代码高亮处是这个 corner case。

### ACcode:
```cpp
#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;
typedef long long ll;
const int N = 1e5 + 5;

ll a[N],b[N],f[N],g[N],c[N],d[N],e[N],h[N];

void solve() {
    int n,q;
    cin >> n >> q;

    for (int i = 0 ; i <= n ; ++i) {
        a[i] = b[i] = f[i] = g[i] = c[i] = d[i] = e[i] = h[i] = 0;
    }

    for (int i = 1 ; i <= n ; ++i) {
        cin >> a[i] >> b[i];
    }
    if (a[1] != 0) {
        while (q--) {
            ll m;
            cin >> m;
            cout << (m ? -1 : 1) << '\n';
        }
        return;
    }
    a[0] = -1;
    for (int i = 1 ; i <= n ; ++i) {
        if (a[i] != (a[i - 1] + 1)) {
            n = i - 1;
            break;
        }
    }

    f[0] = 2e9;
    for (int i = 1 ; i <= n ; ++i) {
        if (b[i] < f[i - 1]) {
            f[i] = b[i],g[i] = i;
        }
        else {
            f[i] = f[i - 1],g[i] = g[i - 1];
        }
    }
    ll now = n,cur = 0,s = 0;
    while (now != 0) {
        c[++cur] = now,d[cur] = f[now] - s;
        s += d[cur];
        now =g[now] - 1;
    }

    for (int i = 1 ; i <= cur ; ++i) {
        e[i] = e[i - 1] + c[i] * d[i];
        h[i] = h[i - 1] + d[i];
    }
    while (q--) {
        ll m;
        cin >> m;
        if (m == 0 || m > e[cur]) {
            cout << -1 << '\n';
        }
        else {
            if (m < c[1]) {
                cout << 2 << '\n';
                continue;
            }
            int pos = upper_bound(e + 1,e + cur + 1,m) - e - 1;
            if (e[pos] == m) cout << h[pos] << '\n';
            else {
                ll ans = h[pos] + (m - e[pos]) / c[pos + 1] + ((m - e[pos]) % c[pos + 1] != 0);
                cout << ans << '\n'; 
            }
        }
    }
}

int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    cout.tie(nullptr);

    int T = 1;
    cin >> T;
    while (T--) {
        solve();   
    }
    return 0;
}
```
