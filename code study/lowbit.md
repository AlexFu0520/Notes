`lowbit` 是编程中一个核心概念，特指 **一个整数二进制表示中，最低位的 1 及其右侧所有 0 组成的数值**。它是位运算的常用工具，广泛用于树状数组（Fenwick Tree）、状态压缩、数论等场景。
### 一、核心定义

对于整数 `x`（通常为非负整数），`lowbit(x)` 的数学表达：

- 若 `x` 的二进制为 `...100...0`（最低位的 1 位于第 `k` 位，从 0 开始计数），则 `lowbit(x) = 2^k`。
- 特殊情况：`x=0` 时，二进制没有 1，故 `lowbit(0) = 0`（无实际意义，通常避免对 0 调用）。
#### 示例（二进制 → 十进制）

|整数 x|二进制表示|最低位的 1 位置|lowbit(x)|
|---|---|---|---|
|6|110|第 1 位（2¹）|2|
|8|1000|第 3 位（2³）|8|
|5|101|第 0 位（2⁰）|1|
|12|1100|第 2 位（2²）|4|
|0|0|无|0|

### 二、计算方法（高效位运算）

`lowbit(x)` 的最简洁实现依赖 **补码特性**，公式为：

```c
lowbit(x) = x & (-x)
```

#### 原理（以补码为基础）

计算机中负数以「补码」存储：`-x = ~x + 1`（按位取反 + 1）。对 `x` 和 `-x` 做按位与（`&`），只有 **最低位的 1** 会被保留（其余位均为 0）：

- 例 1：`x=6`（二进制 `000...0110`）
    - `-x` 的补码：`~6 + 1 = 111...1001 + 1 = 111...1010`
    - `6 & (-6) = 000...0110 & 111...1010 = 000...0010 = 2`（即 lowbit (6)）
- 例 2：`x=5`（二进制 `000...0101`）
    - `-x` 的补码：`~5 + 1 = 111...1010 + 1 = 111...1011`
    - `5 & (-5) = 000...0101 & 111...1011 = 000...0001 = 1`（即 lowbit (5)）

#### 其他语言实现（本质一致）

- Python：`def lowbit(x): return x & (-x)`
- Java：`public static int lowbit(int x) { return x & (-x); }`
- JavaScript：`const lowbit = x => x & (-x);`

### 三、核心应用场景

`lowbit` 的核心价值是 **快速定位和提取最低位的 1**，进而解决高效计数、区间查询等问题：
#### 1. 统计二进制中 1 的个数（汉明重量）

通过循环提取 `lowbit(x)` 并减去，直到 `x=0`，循环次数即为 1 的个数：

```python
def count_ones(x):
    cnt = 0
    while x:
        x -= lowbit(x)  # 每次消去最低位的 1
        cnt += 1
    return cnt

print(count_ones(6))  # 6=110 → 输出 2
print(count_ones(5))  # 5=101 → 输出 2
```

#### 2. 树状数组（Fenwick Tree）核心操作

树状数组的 **单点更新** 和 **前缀和查询** 完全依赖 `lowbit`：

- 前缀和查询（查询 `1~x` 的和）：`sum += tree[x]`，`x -= lowbit(x)`（向上跳）
- 单点更新（给 `x` 加 `val`）：`tree[x] += val`，`x += lowbit(x)`（向下跳）

```python
class FenwickTree:
    def __init__(self, size):
        self.n = size
        self.tree = [0]*(self.n+1)
    
    def update(self, idx, val):
        # 单点更新：idx 位置加 val（idx 从 1 开始）
        while idx <= self.n:
            self.tree[idx] += val
            idx += lowbit(idx)  # 向下跳（覆盖父节点）
    
    def query(self, idx):
        # 前缀和查询：1~idx 的和
        res = 0
        while idx > 0:
            res += self.tree[idx]
            idx -= lowbit(idx)  # 向上跳（累加子节点）
        return res
```

#### 3. 快速判断 2 的幂

若 `x` 是 2 的幂（二进制只有一个 1），则 `lowbit(x) = x`（最低位的 1 就是整个数）：

```python
def is_power_of_two(x):
    return x > 0 and (x & (-x)) == x

print(is_power_of_two(8))  # 8=1000 → True
print(is_power_of_two(6))  # 6=110 → False
```

#### 4. 状态压缩中的位操作

在动态规划（DP）的状态压缩中（如棋盘问题、子集问题），`lowbit` 可快速提取当前状态的「最低有效位」，用于枚举子集或转移状态：

```python
# 枚举 x 的所有非空子集（基于 lowbit）
def enumerate_subsets(x):
    subset = x
    while subset:
        print(subset)  # 输出当前子集（二进制表示的状态）
        subset = x & (subset - 1)  # 消去最低位的 1，得到下一个子集

enumerate_subsets(6)  # 6=110 → 输出 6(110)、4(100)、2(010)
```

### 四、注意事项

1. **负数处理**：若 `x` 为负数，在补码体系下 `x & (-x)` 仍有效（本质是提取负数二进制中最低位的 1），但实际应用中通常对非负整数调用 `lowbit`。
2. **0 的特殊情况**：`lowbit(0) = 0`，但此时无实际意义，需避免传入 0（如树状数组中索引从 1 开始）。
3. **位宽限制**：对于 32 位 / 64 位整数，`lowbit` 不会超出对应位宽（如 32 位 int 的最大值 `2^31-1`，其 lowbit 最多为 `2^30`）。

### 总结

`lowbit` 是位运算的「精髓工具」，核心作用是 **高效提取最低位的 1**，其实现简单（`x & (-x)`），但衍生出树状数组、位计数等高频应用。理解 `lowbit` 的本质（补码特性 + 位与运算），是掌握进阶编程技巧的关键。
