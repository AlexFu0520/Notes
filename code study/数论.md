# 裴蜀定理（贝祖定理）：算法竞赛数论核心基础

裴蜀定理是数论中关于**线性组合与最大公约数（gcd）** 的核心定理，是算法竞赛中解决线性组合、同余方程、不定方程等问题的关键工具。本文将从定义、核心结论、推论、算法实现（扩展欧几里得）、竞赛应用等方面，结合 C++ 代码详细讲解，适配算法竞赛场景。

## 一、核心定义与结论

### 1. 二元形式（基础）

对任意整数 \(a, b\)，存在整数 \(x, y\) 使得：$ax + by = \gcd(a, b))$且 $\gcd(a, b)$是满足该等式的**最小正整数**。
#### 关键推论（竞赛高频考点）

- \(ax + by = c\) 有整数解的充要条件是 $\gcd(a, b) \mid c$（即 c 是 $\gcd(a,b)$ 的倍数）
- 若 a 与 b 互质 $\gcd(a,b)=1$，则存在整数 $(x,y)$使得  $ax + by = 1$（反之亦然，这是判断两数互质的重要方法）；
- 所有满足 $ax + by = c$的整数解，都可由一组特解通过通解公式推导得出。
### 2. 多元形式（扩展）

对任意整数 $a_1, a_2, ..., a_n$，存在整数 $x_1, x_2, ..., x_n$ 使得：$a_1x_1 + a_2x_2 + ... + a_nx_n = \gcd(a_1, a_2, ..., a_n)$ 且该 gcd 是所有线性组合中的最小正整数。

推论：$a_1x_1 + ... + a_nx_n = c$ 有解的充要条件是 $gcd(a_1,...,a_n) \mid c$。
## 二、核心算法：扩展欧几里得算法（exgcd）

裴蜀定理的核心应用是**求线性组合的一组特解**，而扩展欧几里得算法（Extended Euclidean Algorithm）正是实现这一目标的工具。它不仅能计算  $gcd(a,b)$，还能同时求出  $ax + by = \gcd(a,b)$ 的一组整数解  $(x,y)$。

### 1. 算法原理
逆元的定义  $a \cdot x \equiv 1 \pmod{m}$  等价于线性方程  $a \cdot x + m \cdot y = 1$ （y 为整数）。当  $gcd(a, m) = 1$  时，用扩展欧几里得算法求解该方程的 x，即为 a 的逆元（需将 x 调整为非负）。
### 2.exgcd代码

```cpp
int exgcd(int a,int b,int& x,int& y) {
	if (b == 0) {
		x = 1;
		y = 0;
		return a;
	}
	int d = exgcd(b,a%b,x,y);
	int tmp = x;
	x = y;
	y = tmp - a / b * y;
	return d;
}
```

### 2. 逆元存在的充要条件

由**裴蜀定理**可知：$a \cdot x + m \cdot y = 1$ 有整数解的充要条件是  $gcd(a, m) = 1$（即 a 与 m 互质）。因此，**a 在模 m 下存在逆元的充要条件是 $gcd(a, m) = 1$。

#### 推论

- 若 m 是质数，则对于所有 $1 \leq a < m$，a 与 m 互质，因此所有 a 都存在模 m 的逆元（这是费马小定理的应用基础）。
- 若 m 不是质数，则仅当 a 与 m 互质时，逆元存在（如 $a=2, m=4$ , $gcd(2,4)=2 \neq 1$，逆元不存在）。

## 三、逆元的四种核心求法（C++ 实现）

竞赛中逆元的求法需根据 m 的性质（是否为质数、是否需要预处理多个逆元）选择，以下是四种高频方法：

### 方法 1：扩展欧几里得算法（通用性最强）

#### 原理

逆元的定义 $a \cdot x \equiv 1 \pmod{m}$ 等价于线性方程  $a \cdot x + m \cdot y = 1$（y 为整数）。当 $gcd(a, m) = 1$时，用扩展欧几里得算法求解该方程的 x，即为 a 的逆元（需将 x 调整为非负）。
#### C++ 实现

```cpp
#include <iostream>
using namespace std;
typedef long long ll;

// 扩展欧几里得算法：返回 gcd(a,m)，并通过引用传出 ax + my = 1 的解 x（逆元）
ll exgcd(ll a, ll m, ll &x, ll &y) {
    if (m == 0) {
        x = 1, y = 0;
        return a;
    }
    ll g = exgcd(m, a % m, x, y);
    ll temp = x;
    x = y;
    y = temp - (a / m) * y;
    return g;
}

// 求 a 在模 m 下的逆元（存在返回逆元，不存在返回 -1）
ll mod_inverse_exgcd(ll a, ll m) {
    ll x, y;
    ll g = exgcd(a, m, x, y);
    if (g != 1) return -1; // 不互质，逆元不存在
    return (x % m + m) % m; // 调整为非负逆元
}

int main() {
    ll a = 5, m = 7; // 5 和 7 互质，逆元存在
    ll inv = mod_inverse_exgcd(a, m);
    if (inv == -1) cout << "逆元不存在" << endl;
    else cout << a << "的逆元（模" << m << "）是：" << inv << endl; // 输出 3（5*3=15≡1 mod7）
    return 0;
}
```
#### 适用场景

- m 可为任意正整数（质数 / 合数均可）；
- 适合单次查询单个逆元（时间复杂度 $O(\log \min(a,m)$)。

### 方法 2：费马小定理（仅适用于 m 为质数）

#### 原理

若 m 是质数，且 a 与 m 互质，则： $a^{m-1} \equiv 1 \pmod{m}$两边同时除以 a（因 a 与 m 互质，除法等价于乘逆元），得： $a^{m-2} \equiv a^{-1} \pmod{m}$因此，a 的逆元为 $a^{m-2} \mod m$。

<img src = images/费马小定理.png> 

#### C++ 实现（快速幂优化）

```cpp
#include <iostream>
using namespace std;

typedef long long ll;

// 快速幂：计算 (base^exp) mod modu
ll quick_pow(ll base, ll exp, ll modu) {
    ll res = 1;
    base %= modu; // 避免base过大
    while (exp > 0) {
        if (exp & 1) res = (res * base) % modu;
        base = (base * base) % modu;
        exp >>= 1;
    }
    return res;
}

// 费马小定理求逆元：m必须是质数，a与m互质（m为质数时a≠0 mod m即可）
ll mod_inverse_fermat(ll a, ll m) {
    if (a % m == 0) return -1; // a与m不互质（m为质数时，a是m的倍数），逆元不存在
    return quick_pow(a, m - 2, m);
}

int main() {
    ll a = 5, m = 7; // m是质数
    ll inv = mod_inverse_fermat(a, m);
    if (inv == -1) cout << "逆元不存在" << endl;
    else cout << a << "的逆元（模" << m << "）是：" << inv << endl; // 输出3
    return 0;
}
```
#### 适用场景

- m 是质数（如竞赛中常见的 \(1e9+7\)、998244353）；
- 单次查询或批量查询（快速幂时间复杂度 \(O(\log m)\)），效率高于扩展欧几里得。

### 方法 3：欧拉定理（通用但需计算欧拉函数）

#### 原理

欧拉函数  $\varphi(m)$ 表示 \($1 \sim m-1$\) 中与 m 互质的整数个数。若 a 与 m 互质，则：$a^{\varphi(m)} \equiv 1 \pmod{m}$因此，a 的逆元为  $a^{\varphi(m)-1} \mod m$。

#### 关键步骤

1. 计算欧拉函数 $\varphi(m)$（需对 m 质因数分解）；
2. 用快速幂计算 $a^{\varphi(m)-1} \mod m$。

<img src = images/欧拉函数.png>
#### C++ 实现

```cpp
#include <iostream>
#include <unordered_set>
using namespace std;

typedef long long ll;

// 计算欧拉函数 phi(m)
ll euler_phi(ll m) {
    ll res = m;
    // 质因数分解
    for (ll i = 2; i * i <= m; ++i) {
        if (m % i == 0) {
            while (m % i == 0) m /= i;
            res -= res / i; // 欧拉函数公式：res = res * (1 - 1/i)
        }
    }
    if (m > 1) res -= res / m; // 剩余质因子
    return res;
}

ll quick_pow(ll base, ll exp, ll modu) {
    ll res = 1;
    base %= modu;
    while (exp > 0) {
        if (exp & 1) res = (res * base) % modu;
        base = (base * base) % modu;
        exp >>= 1;
    }
    return res;
}

// 欧拉定理求逆元：a与m互质时存在逆元
ll mod_inverse_euler(ll a, ll m) {
    ll g = __gcd(a, m);
    if (g != 1) return -1;
    ll phi = euler_phi(m);
    return quick_pow(a, phi - 1, m);
}

int main() {
    ll a = 5, m = 8; // m=8，phi(8)=4（1,3,5,7与8互质）
    ll inv = mod_inverse_euler(a, m);
    if (inv == -1) cout << "逆元不存在" << endl;
    else cout << a << "的逆元（模" << m << "）是：" << inv << endl; // 输出5（5*5=25≡1 mod8）
    return 0;
}
```

#### 适用场景

- m 为任意正整数（质数 / 合数）；
- 缺点：计算欧拉函数需质因数分解，时间复杂度 \(O(\sqrt{m})\)，比扩展欧几里得稍慢，竞赛中不如前两种常用。

### 方法 4：递推预处理逆元（适用于 $1 \sim n$ 批量查询）

#### 原理

当需要计算 $1 \sim n$ 中所有数的逆元（如组合数批量计算）时，递推公式效率最高（时间复杂度 $O(n)$。前提：m 是质数（竞赛中常用场景），递推公式如下：

- 边界： $1^{-1} \equiv 1 \pmod{m}$
- 递推式：$i^{-1} \equiv (m - m/i) \cdot (m \mod i)^{-1} \pmod{m}$
#### 推导

设 $m = q \cdot i + r$（$(0 < r < i$），则 $q = m/i$，$r = m \mod i$，
因此： $q \cdot i + r \equiv 0 \pmod{m}$两边乘  $i^{-1} \cdot r^{-1}$，
得：$q \cdot r^{-1} + i^{-1} \equiv 0 \pmod{m}$ 
整理得： $i^{-1} \equiv -q \cdot r^{-1} \equiv (m - q) \cdot r^{-1} \pmod{m}$ 即 $i^{-1} = (m - m/i) \cdot (m \mod i)^{-1} \mod m$ 。

#### C++ 实现

```cpp
#include <iostream>
#include <vector>
using namespace std;

typedef long long ll;

const int MAXN = 1e6 + 10; // 预处理1~1e6的逆元
ll inv[MAXN];

// 递推预处理1~n的逆元（m必须是质数）
void pre_inv(ll n, ll m) {
    inv[1] = 1;
    for (ll i = 2; i <= n; ++i) {
        inv[i] = (m - m / i) * inv[m % i] % m;
    }
}

int main() {
    ll m = 1e9 + 7; // 质数
    ll n = 1e5;     // 预处理1~1e5的逆元
    pre_inv(n, m);
    cout << "5的逆元（模" << m << "）是：" << inv[5] << endl; // 输出200000001（5*200000001=1e9+5≡1 mod1e9+7）
    cout << "10的逆元（模" << m << "）是：" << inv[10] << endl; // 输出100000001
    return 0;
}
```

#### 适用场景
- m 是质数（如 \(1e9+7\)）；
- 需要批量查询 $1 \sim n$ 的逆元（如组合数计算、动态规划中的模运算），效率极高。

唯一分解定理：任一大于1的整数都能表示成质数的乘积

## 四、注意事项与易错点

1. **处理负数**：gcd 本身是非负的，因此计算时需对 \(a, b\) 取绝对值，避免符号影响结果；
2. **解的多样性**：裴蜀定理的解不唯一，竞赛中可能需要求 “最小非负解”“正整数解” 等，需利用通解公式调整；
3. **数据范围**：当 \(a, b\) 是大数（如 $10^{18}$）时，需用 `long long` 避免溢出，递归版 exgcd 可能栈溢出，建议用迭代版；
4. **库函数使用**：C++17 及以上提供 `std::gcd`（需包含 `<numeric>`），但部分竞赛环境可能不支持，建议自备 gcd 或 exgcd 实现。

## 五、经典例题（牛客 / 洛谷高频题）

### 例题 1：洛谷 P1082 [NOIP2012 提高组] 同余方程

**题目描述**：求关于 x 的同余方程  $ax \equiv 1 \pmod{m}$ 的最小正整数解。**解法**：本质是求 a 在模 m 下的逆元，直接用扩展欧几里得算法。
#### 代码实现

```cpp
#include <iostream>
using namespace std;

typedef long long ll;

ll exgcd(ll a, ll b, ll &x, ll &y) {
    if (b == 0) {
        x = 1, y = 0;
        return a;
    }
    ll g = exgcd(b, a % b, x, y);
    ll temp = x;
    x = y;
    y = temp - (a / b) * y;
    return g;
}

int main() {
    ll a, m, x, y;
    cin >> a >> m;
    ll d = exgcd(a, m, x, y);
    // 因题目保证有解（输入满足 gcd(a,m)=1），直接求最小正解
    x = (x % m + m) % m;
    cout << x << endl;
    return 0;
}
```

### 例题 2：牛客 NC14522 硬币问题

**题目描述**：有面值为 a 和 b 的两种硬币，求不能用它们凑出的最大金额（即 “Frobenius 数”）。**前提**：a 和 b 互质，否则不存在最大不能凑出的金额（无数个）。**结论**：最大不能凑出的金额为 $ab - a - b$。
#### 代码实现

```cpp
#include <iostream>
#include <algorithm>
using namespace std;

int main() {
    int a, b;
    cin >> a >> b;
    int d = __gcd(a, b);
    if (d != 1) {
        cout << "无数个不能凑出的金额" << endl;
    } else {
        cout << a * b - a - b << endl;
    }
    return 0;
}
```

## 六、总结

裴蜀定理是算法竞赛数论模块的 “基石”，核心围绕 “线性组合与 gcd 的关系” 展开。掌握扩展欧几里得算法的实现，能解决大部分与线性组合、不定方程、同余方程相关的题目。竞赛中需重点关注：

1. 解的存在性判断 ($c \mod \gcd(a,b) == 0$)；
2. 特解与通解的计算；
3. 逆元、同余方程等衍生问题。

建议通过大量例题（如洛谷 P1082、P1516，牛客 NC14522）巩固应用，确保能快速调用 exgcd 并灵活处理边界情况（负数、大数、最小解等）。
