## 文件与内容规范

### 1. 每日任务 / 日记（Daily Task）规范

- 命名规则：统一采用 `YYYY-MM-DD.md` 格式，单日单文件，按年度 - 月度维度归档
- 核心内容模块（可按需增减）：
    
    - 当日概览：天气、核心完成事项、当日状态
    - 任务完成情况：当日计划完成度、未完成事项与原因分析
    - 学习与成长：当日学习内容、核心收获、认知升级点
    - 反思与复盘：当日不足、可优化点、改进方案
    - 明日计划：次日核心待办事项
    

### 2. 算法模板规范

- 按算法专题分类归档，单专题单文件，内容包含模板代码、适用场景、复杂度分析、使用示例与注意事项
- 模板代码保证可直接复用，标注核心考点与踩坑点，同时适配算法竞赛与日常刷题场景

### 3. 题目记录规范

- 单题单文件，命名格式为 `YYYY-MM-DD-题目名称.md`
- 核心必填模块：
    
    - 基础信息：题目来源、难度、考察知识点、首次做题日期
    - 题目原文与配套配图
    - 解题思路与多解法对比分析
    - 完整可运行代码 / 标准解题步骤
    - 复杂度分析与核心考点拆解
    - 错题复盘（做错时必填）：错误原因、关键卡点、避坑方案
    - 总结与同类题拓展
    

### 4. 课程与代码学习笔记规范

- 课程笔记按课程 / 章节分类，梳理核心知识点、重点难点、随堂案例，保证知识体系的连贯性与完整性
- 代码学习笔记配套源码、详细注释、原理解读与运行结果，相关截图与配图统一存放于 `code study/images` 目录，使用相对路径引用
