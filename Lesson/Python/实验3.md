print(df_fill_forward)

print("\n均值填充缺失值后的数据：")
print(df_fill_mean)

df_fill_interp = df_yq.interpolate()

print(df_fill_interp)

### 七、思考题

### 1. 比较不同缺失值处理方法的优缺点
#### 一、删除法（`dropna()`）
**核心逻辑**：直接删除包含缺失值的行 / 列，是最基础的缺失值处理方式。
- **优点**：   
    1. 操作极简，仅需一行代码即可完成，无需复杂计算；
    2. 完全保留数据原始性，不会引入人为误差；
    3. 适合缺失值占比极低（如 < 5%）且缺失随机的场景。
- **缺点**：
    1. 丢失数据样本，若缺失值占比高（如 > 10%），会导致样本量不足、分析结果偏差；
    2. 对时序数据（如疫情每日数据）破坏性大：删除某行相当于丢失某一天的记录，破坏时间连续性；
    3. 若缺失非随机（如某地区持续缺数据），删除后会导致该维度数据不完整。
- **适用场景**：缺失值占比极低、数据量充足、缺失随机的非时序数据（如电商 action 列缺失可直接删）。
#### 二、前向 / 后向填充（`fillna(method='ffill'/'bfill')`）
**核心逻辑**：前向填充用前一行值填充缺失值，后向填充用后一行值填充，是时序数据专属方法。
- **优点**：
    1. 保留所有样本和时序连续性（疫情数据不会丢失日期）；
    2. 执行效率高，无需统计计算，贴合 “相邻时间数据关联性强” 的特点；
    3. 代码简单，新手易掌握。
- **缺点**：
    1. 连续多行缺失时，会重复填充同一个值，放大误差（如某地区连续 3 天缺新增确诊，会一直用第一天的值）；
    2. 仅适用于时序 / 有序数据，对电商用户行为这类无顺序数据无意义；
    3. 仅 “机械填充”，无法反映数据真实趋势。
- **适用场景**：时序数据（疫情、股价）、缺失值为零星单行的场景。
#### 三、均值 / 中位数 / 众数填充（`fillna(mean/median/mode)`）

**核心逻辑**：用数值列的均值 / 中位数（数值型）、众数（分类型）填充，是通用型填充方法。
- **优点**：
    1. 保留所有样本，不会丢失数据；
    2. 计算简单，能反映数据集中趋势，适合缺失值随机分布的场景；
    3. 分类数据（如电商 action 列）用众数填充更贴合实际（用最频繁的行为填充）。
- **缺点**：    
    1. 拉低数据方差，使数据 “趋同”，掩盖真实波动（如疫情数据用均值填充突发增长的缺失值，会抹平峰值）；
    2. 均值易受异常值影响（如疫情某天大数异常，均值偏高），中位数更稳健但仍无时序特征；
    3. 完全忽略数据的时序 / 关联特征，仅用全局统计值填充。
- **适用场景**：非时序数值数据（如电商价格）、缺失值随机且无明显趋势的场景。
#### 四、插值填充（`interpolate()`）

**核心逻辑**：通过线性 / 多项式插值，根据缺失值前后的有效数据计算 “合理值” 填充，是时序数据最优方法之一。
- **优点**：
    1. 保留时序连续性，填充值贴合数据真实趋势（疫情数据会按前后天数的增长 / 下降趋势计算）；
    2. 连续缺失值也能生成渐变值，误差远小于前向填充；
    3. 支持多种插值方式，适配不同趋势的时序数据。
- **缺点**：
    1. 计算开销略高于前向填充 / 均值填充；
    2. 仅适用于有序 / 时序数据，对离散数据无效；
    3. 若缺失值前后数据异常，插值结果也会偏差（但概率远低于均值填充）。
- **适用场景**：时序数据（实验中疫情数据优先用此方法）、缺失值有连续趋势的场景
#### 六、核心方法对比表

| 处理方法       | 核心优点       | 核心缺点         | 优先适用场景      |
| ---------- | ---------- | ------------ | ----------- |
| 删除法        | 简单无误差      | 丢失数据、破坏时序    | 缺失少、数据足、非时序 |
| 前向 / 后向填充  | 保留时序、执行快   | 连续缺失误差大      | 时序数据、零星缺失   |
| 均值 / 中位数填充 | 通用、保留样本    | 抹平方差、无趋势性    | 非时序、随机缺失    |
| 插值填充       | 贴合时序趋势、误差小 | 计算略复杂、仅适有序数据 | 时序数据、有趋势的缺失 |

### 2. 总结数据清洗的常见步骤
#### 步骤 1：数据读取与初步探索（基础前提）

**核心动作**：
- 读取原始数据（CSV/JSON 等格式，如`pd.read_csv()`读取疫情 / 电商数据）；
- 探索数据基本特征：查看数据形状（`shape`）、前 N 行（`head()`）、数据类型（`info()`）、描述统计（`describe()`）；
- 初步识别问题：快速定位缺失值、异常值、重复值的大致分布。
#### 步骤 2：重复值处理（优先解决）

**核心动作**：
- 检测重复值：用`duplicated().sum()`统计完全重复行数量；
- 删除重复值：
    - 完全重复行：`drop_duplicates()`直接删除（如电商数据先清理全重复记录）；
    - 关键字段重复：按核心维度（如电商的`user_id+product_id+action`）去重，保留最后 / 第一条（`keep='last'`）。
#### 步骤 3：缺失值处理（核心环节）

**核心动作**：
- 统计缺失值：`isnull().sum()`明确各列缺失比例；
- 选择适配的处理方法：
    - 删除法：缺失值占比极低（<5%）或无业务价值的列（如电商`action`列缺失直接删除）；
    - 填充法：缺失值占比高或有价值的列（时序数据用插值 / 前向填充，数值列用均值 / 中位数，分类列用众数）。
#### 步骤 4：异常值处理（关键优化）

**核心动作**：
- 识别异常值：
    - 数值型：通过描述统计、四分位数（IQR）、业务阈值（如电商价格 > 5000）识别；
    - 逻辑型：通过业务规则（如价格为负、日期格式错误）识别；
- 处理异常值：
    - 删除法：异常值少且无意义（如电商负价格直接删除）；
    - 修正法：异常值多且有规律（如电商超高价格替换为 99 分位数）；
#### 步骤 5：数据类型转换（适配分析）

**核心动作**：
- 转换为目标类型：
    - 日期列：`pd.to_datetime()`（疫情 “日期” 列转 datetime，支持时序分析）；
    - 数值列：`astype('float64')`（确保价格、确诊数为数值型，支持计算）；
    - 分类列：`astype('category')`（如疫情 “地区” 列，节省内存且适配分组统计）。
#### 步骤 6：特征提取 / 数据标准化（可选，进阶）

**核心动作**：
- 特征提取：从原始数据衍生有价值的新特征（如电商按`user_id`统计 “行为次数”“平均消费”）；
- 标准化：数值列统一量纲（如价格归一化到 0-1，适配建模需求）。
#### 步骤 7：数据保存（收尾）

**核心动作**：
- 将清洗后的干净数据保存为新文件（如`df.to_csv('ecommerce_data_clean.csv')`），避免重复清洗。
