### 七、思考题

### 1. 总结 Python 基本数据类型的特点

Python 的基本数据类型是编程的基础，主要分为**数值型、字符串型、布尔型**（补充空类型 NoneType），核心特点如下：
#### 1. 数值类型（int /float/complex）
- **整数（int）**：
    - 无长度限制：可存储任意大小的整数（如`10`、`-5`、`1000000000000000`），无需担心溢出；
    - 不可变：赋值后无法修改原整数，重新赋值会生成新对象；
    - 支持运算：算术运算（+/-/*/÷）、位运算（&/|/^）、比较运算（>/</==）等。
- **浮点数（float）**：
	- 存储小数（如`3.14`、`-0.5`），也可表示科学计数法（如`1.23e4`即 12300）；
    - 存在精度误差：因二进制浮点存储规则，`0.1 + 0.2 ≠ 0.3`（结果为`0.30000000000000004`）；
    - 不可变，支持与整数混合运算（结果自动转为 float）。
- **复数（complex）**：
    - 格式为`a + bj`（`a`实部、`b`虚部，如`2+3j`），专为科学计算设计；
    - 支持复数专属运算（如取实部`(2+3j).real`、虚部`(2+3j).imag`）。
#### 2. 字符串类型（str）
- **定义灵活**：可通过单引号`'abc'`、双引号`"abc"`、三引号`'''多行字符串'''`定义，三引号支持换行；
- **不可变**：无法直接修改字符串中的单个字符（如`s[0] = 'A'`会报错），需通过切片、拼接生成新字符串；
- **核心操作**：支持索引（`s[0]`取第一个字符）、切片（`s[0:3]`取前 3 个字符）、拼接（`s1 + s2`）、重复（`s*3`）；
- **丰富方法**：内置`strip()`（去空格）、`upper()`（转大写）、`replace()`（替换）等方法，满足文本处理需求。
#### 3. 布尔类型（bool）
- **仅有两个值**：`True`（真，本质等价于整数 1）、`False`（假，本质等价于整数 0）；
- **核心用途**：用于条件判断（`if`/`while`）、逻辑运算（`and`/`or`/`not`）；
- **类型转换规则**：“空 / 零” 值（如`0`、`""`、`[]`）转`False`，非空 / 非零值（如`1`、`"abc"`）转`True`。
#### 4. 空类型（NoneType）
- 唯一值为`None`，代表 “无”“空”，区别于`0`或空字符串；
- 常用于函数返回值（无明确返回值时默认返回`None`）、变量初始化（表示暂未赋值）。
#### 通用特性
- 所有基本类型均为**不可变类型**：修改值会创建新对象，而非修改原对象；
- 可通过`type(变量)`查看类型，`isinstance(变量, 类型)`判断类型归属；
- 支持显式类型转换（如`int("123")`、`str(123)`），但转换需符合规则（如`int("abc")`会报错）。

### 2. 思考：在 Jupyter Notebook 中使用有哪些优势？
#### 1. 交互式分段执行，降低新手调试门槛
这是最核心的优势。传统.py 脚本需要一次性运行全部代码，若某一行出错则整个脚本终止；而 Jupyter 将代码拆分为**单元格**，可逐段运行（如实验中 “变量定义”“类型转换”“BMI 计算” 各占一个单元格），每段代码运行后即时输出结果，出错仅需修改对应单元格，无需从头执行。例如编写 BMI 计算器时，可先单独调试 “身高体重输入” 单元格，确认输入格式无误后，再运行 “BMI 计算” 和 “状态判断” 单元格，新手能快速定位 “类型转换错误”“公式写错” 等问题，大幅降低调试难度。
#### 2. 代码与文档一体化，一站式完成实验报告
Jupyter 支持**代码 + Markdown 文本**混合排版：在代码单元格之间，可插入实验步骤说明、注释、结论（如用 Markdown 写 “练习 3：数值运算 —— 验证算术 / 比较运算符规则”），还能插入公式（如 BMI 计算公式体重身高）、截图（运行结果截图），无需在 “代码编辑器” 和 “Word 文档” 之间来回切换。实验完成后，可直接将 Notebook 导出为 Word/PDF/HTML 格式，省去 “复制代码→粘贴到文档→排版” 的繁琐步骤，让实验报告的整理更高效。
#### 3. 变量复用与实时交互，贴合实验推进逻辑
实验中代码是逐步推进的（如先定义变量`a=10`，再用`a`做算术运算），Jupyter 的变量可跨单元格复用 —— 前一个单元格定义的变量 / 函数，后续单元格无需重新定义即可直接调用，符合 “从基础语法到综合案例” 的实验逻辑。此外，Jupyter 支持实时交互，比如 BMI 计算器的`input()`函数，可在单元格中直接输入身高 / 体重，无需像传统脚本那样在终端单独运行，交互过程更直观，也便于观察输入错误（如输入非数字字符）时的报错信息。
#### 4. 环境适配性强，易安装、易分享
Jupyter 与 Anaconda 环境无缝兼容，仅需`conda install jupyter`即可安装，无需额外配置 Python 解释器路径，新手能快速上手；实验完成后，可直接保存.ipynb 文件（包含代码、运行结果、文本说明），分享给他人后，对方只需打开文件就能复现实验结果，无需担心 “环境不一致导致代码运行失败”；若需清理实验中的变量冲突，还可一键重启内核，重新开始测试，资源管理更灵活。
