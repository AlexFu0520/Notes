### 七、思考题

### 1. 比较迭代和递归两种实现方式的优缺点
#### 一、迭代（循环实现）

迭代通过`for`/`while`循环重复执行代码块，手动控制循环条件和状态，是最基础的流程控制方式。
#### 优点
1. **性能更优**
    无函数调用的额外开销（如栈帧创建、参数传递、返回地址保存），执行效率高。例如计算`1000!`，迭代可直接循环计算，而递归会因函数调用栈的开销导致速度显著变慢。
2. **内存可控，无溢出风险**
    仅占用固定的内存空间（如循环变量、中间结果变量），不受 “最大递归深度” 限制。Python 默认递归深度约 1000 层，若计算`n=1000`的阶乘，递归会抛出`RecursionError`，但迭代可正常执行。
3. **逻辑直观（简单场景）**
    对于线性流程的问题（如素数筛选、九九乘法表、批量成绩处理），迭代的执行步骤清晰，符合 “从初始状态逐步推进” 的思维习惯，易读易调试。
4. **无重复计算
    如斐波那契迭代实现通过列表存储中间值，每个数仅计算一次；而递归实现会重复计算`f(n-2)`、`f(n-3)`等子问题（如`f(5)`需计算`f(4)+f(3)`，`f(4)`又需计算`f(3)+f(2)`，`f(3)`被重复计算）。
#### 缺点
1. **逻辑复杂度高（复杂场景）**
    对于分治类、递归结构的问题（如树遍历、斐波那契 / 阶乘的数学定义），迭代需要手动维护中间状态（如斐波那契的列表、树遍历的手动栈），代码冗余且可读性下降。
2. **贴近问题本质性弱**
    迭代需要将 “分治思想” 转化为 “循环 + 状态变量”，与阶乘`n! = n × (n-1)!`、斐波那契`F(n)=F(n-1)+F(n-2)`的数学定义脱节，代码逻辑不如递归贴合问题本身。
#### 二、递归（函数自调用）
递归通过函数调用自身，将原问题分解为规模更小的同类型子问题，直到触发 “基线条件”（终止条件）后回溯求解。
#### 优点
1. **代码简洁，开发效率高**
    直接对应分治问题的数学定义，代码行数少、逻辑清晰。例如阶乘递归仅需几行代码
    无需手动维护循环变量，快速实现核心逻辑。
2. **逻辑直观（分治 / 递归结构问题)**
    处理树、图、分形等递归结构的问题时，递归无需手动模拟 “遍历状态”，逻辑更贴近问题本质（如二叉树的后序遍历，递归仅需 “左→右→根” 的逻辑，迭代需手动维护栈）。
#### 缺点
1. **性能开销大**
    每次函数调用都会创建栈帧（保存参数、局部变量、返回地址），带来额外的内存和时间开销，效率远低于迭代。
2. **栈溢出风险**
    受编程语言的递归深度限制（Python 默认约 1000 层），超出限制会直接报错。例如实验中若用递归计算`n=2000`的阶乘，会立即触发`RecursionError`。
3. **调试难度高**
    递归调用链长，出错时需回溯多层栈帧才能定位问题，而迭代的错误仅出现在循环体内，易排查。
4. **重复计算（未优化）**
    如斐波那契递归实现会重复计算大量子问题（计算`F(10)`需重复计算`F(8)` 2 次、`F(7)` 3 次……），时间复杂度为`O(2ⁿ)`，而迭代的时间复杂度仅为`O(n)`。
#### 三、核心维度对比表

| 对比维度  | 迭代               | 递归                                  |
| ----- | ---------------- | ----------------------------------- |
| 时间性能  | 高（无函数调用开销）       | 低（栈帧 + 重复计算开销）                      |
| 内存占用  | 低且可控（固定空间）       | 高（调用栈占用）                            |
| 栈溢出风险 | 无                | 有（受递归深度限制）                          |
| 代码简洁性 | 简单问题直观，复杂问题冗余    | 分治问题简洁，贴合数学定义                       |
| 调试难度  | 低（循环流程易追踪）       | 高（调用链长）                             |
| 适用场景  | 简单循环、大数计算、性能敏感场景 | 分治问题（如阶乘 / 斐波那契）、递归结构（树 / 图）、快速原型开发 |

#### 四、结合实验案例的总结
以实验中的阶乘、斐波那契实现为例：
- 递归更适合**快速验证逻辑**（如实验中快速实现阶乘的数学定义），但无法处理大数、性能差；
- 迭代更适合**实际工程场景**（如批量成绩处理、素数筛选），性能稳定、无溢出风险，是生产环境的首选；
- 若需用递归优化（如斐波那契的记忆化递归），可平衡简洁性和性能，但会增加代码复杂度。


### 2. 思考：如何优化素数筛选算法？
#### 一、经典优化：埃拉托斯特尼筛法（埃氏筛，Sieve of Eratosthenes）

逐个判断法的核心问题是 “重复判断因子”，而埃氏筛通过**批量标记非素数**的思路，将筛选 1-N 素数的时间复杂度降至`O(N log log N)`，是小范围素数筛选的最优选择。
#### 原理：
1. 初始化长度为`N+1`的布尔数组`is_prime`，初始值全为`True`（假设所有数都是素数）；
2. 将`is_prime[0]`和`is_prime[1]`设为`False`（0 和 1 不是素数）；
3. 从最小素数 2 开始遍历到`√N`：
    - 若`i`是素数（`is_prime[i] == True`），则将`i`的所有倍数（从`i*i`开始，步长`i`）标记为`False`（这些倍数是合数）；
4. 最终数组中值为`True`的索引即为素数（无需逐个判断）。
#### 埃氏筛实现代码（筛选 1-100 的素数）：

```python
def sieve_of_eratosthenes(n):
    """埃拉托斯特尼筛法，筛选1-n的所有素数"""
    if n < 2:
        return []
    # 初始化素数标记数组
    is_prime = [True] * (n + 1)
    is_prime[0] = is_prime[1] = False  # 0和1不是素数
    
    # 遍历到sqrt(n)即可（更大的数已被小素数标记）
    for i in range(2, int(n**0.5) + 1):
        if is_prime[i]:  # i是素数，标记其所有倍数为非素数
            # 从i*i开始标记（更小的倍数已被之前的素数标记）
            is_prime[i*i : n+1 : i] = [False] * len(is_prime[i*i : n+1 : i])
    
    # 收集所有素数（索引为True的数）
    primes = [num for num, flag in enumerate(is_prime) if flag]
    return primes

# 测试筛选1-100的素数
primes = sieve_of_eratosthenes(100)
print("1-100之间的素数（埃氏筛）：")
print(primes)
print(f"共找到 {len(primes)} 个素数")
```
#### 埃氏筛的额外优化：

- 仅标记奇数的倍数：初始化时将偶数（除 2 外）直接标记为`False`，后续仅处理奇数，节省一半空间和时间；
- 从`i*i`开始标记：`i`的更小倍数（如`i*2`、`i*3`）已被更小的素数（2、3）标记过，无需重复操作。
#### 二、进阶优化：欧拉筛（线性筛，Sieve of Euler）

埃氏筛仍存在 “重复标记” 问题（例如 12 会被 2 和 3 都标记），欧拉筛通过**保证每个合数仅被其最小质因子标记**，将时间复杂度降至`O(N)`（线性），是大范围素数筛选的最优方案。
#### 原理：
1. 初始化两个数组：`is_prime`（标记是否为素数）、`primes`（存储已找到的素数）；
2. 遍历 2 到 N 的每个数`i`：
    - 若`is_prime[i]`为`True`，说明`i`是素数，加入`primes`列表；
    - 遍历`primes`中的每个素数`p`：    
        - 标记`i*p`为非素数；
        - 若`i % p == 0`（说明`p`是`i`的最小质因子），立即终止循环（避免后续更大的素数重复标记）。

#### 欧拉筛实现代码（筛选 1-100 的素数）：

```python
def sieve_of_euler(n):
    """欧拉筛（线性筛），筛选1-n的所有素数"""
    if n < 2:
        return []
    is_prime = [True] * (n + 1)
    is_prime[0] = is_prime[1] = False
    primes = []  # 存储已找到的素数
    
    for i in range(2, n + 1):
        if is_prime[i]:
            primes.append(i)
        # 标记i*p为非素数，p是已找到的素数
        for p in primes:
            if i * p > n:  # 超出范围，停止
                break
            is_prime[i * p] = False
            if i % p == 0:  # p是i的最小质因子，终止循环避免重复标记
                break
    return primes

# 测试筛选1-100的素数
primes = sieve_of_euler(100)
print("1-100之间的素数（欧拉筛）：")
print(primes)
print(f"共找到 {len(primes)} 个素数")
```
