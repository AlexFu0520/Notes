## 问题三分析

1. 该论文在处理原始数据时，将所有的公交线路（如L001至L800+）归纳为三种基本类型，并分别制定了抽象规则。
	 - 传统的上下行原路返回型线路，这类线路虽然经过相同的站点序列，但由于其方向性不同，在有向图中必须被抽象为两条独立的方向相反的路径。
	 - 环行线路，论文为了确保任意两站点间距离计算的精确性，将其进一步拆分为多条单向行驶路径，以模拟真实的环形运转逻辑。
	 - 上下行经过站点不同的复杂线路，这类线路被严格视为两条独立的有向路径进行处理 。
2. 在面对3957个站点的海量数据时，传统的实时路径搜索算法（如Dijkstra算法直接作用于原始线网）在处理大规模用户并发查询时，往往会面临极大的计算延迟 。所以应该采取“空间换取时间”策略，即预先建立一个“公交直达数据库Q”。
3. 问题的第三问引入了一个全新的变量：所有站点之间的步行时间 。这一要求的加入，使得传统的单纯依赖交通工具的路径规划演变为多模态（Multimodal）路径规划。用户不仅可以坐车，还可以步行一小段距离前往邻近车站以换取更优的线路选择 。
4. 在原有的公交、地铁网络图层之上，叠加了一个完全由步行时间定义的权重层。此时，图 $G = (V, E, W)$ 中的边集 $E$ 包含了两种属性完全不同的弧：
	- 第一类是乘车弧 $E_{ride}$，其权重 $w_{ride}$ 代表了乘坐公交或地铁的时间和经济成本。
	- 第二类是步行弧 $E_{walk}$，其权重 $w_{walk}$ 代表了在站点间步行的时间成本
	- 论文在算法实现中引入了阈值 $\omega$，即只考虑步行时间在可接受范围内的弧 。
5. 在模型III中，决策变量不再是单一的乘车动作。论文引入了 $x_{ij}$ 与 $y_{ij}$ 两个0-1变量：
	-  $x_{ij}$：如果用户选择从站点 $i$ 乘车前往站点 $j$，则 $x_{ij}=1$，否则为0。
    -  $y_{ij}$：如果用户选择从站点 $i$ 步行前往站点 $j$，则 $y_{ij}=1$，否则为0 。
## 数学分析(多目标)：
1. 总目标：行程总时间最小化$$\min Z_1 = \sum_{(i,j) \in E} (t_{ij}^1 x_{ij} + t_{ij}^2 y_{ij}) + \sum T_{wait} + \sum T_{transfer}$$其中，$t_{ij}^1$ 为从 $i$ 到 $j$ 的乘车时间，$t_{ij}^2$ 为从 $i$ 到 $j$ 的步行时间。还考虑了在途时间、起始站等车时间（公交3分钟，地铁2分钟）以及换乘站点的额外步行和等待损耗。
2. 总费用最低化目标：$$\min Z_2 = \sum_{(i,j) \in E} P_{ij} x_{ij}$$步行不产生费用，而乘车费用 $P_{ij}$ 则需根据车辆属性 $q_{ij}$（单一票制或分段计价）进行累加计算。对于分段计价车辆，其单段费用根据经过的站数 $s_{ij}$ 设定

3. 转乘车辆始发数最大化（舒适度目标）：$$\max Z_3 = \sum_{(i,j) \in E} f_{ij} x_{ij}$$$f_{ij}$ 是一个人性化权重，如果换乘点是某线路的始发站，则该项对目标函数的贡献增加，从而在同等时间内优先推荐有座位的方案 。
4. 换乘站点负载量最小化（流畅度目标）：$$\min Z_4 = \sum_{(i,j) \in E} r_{i} (x_{ij} + y_{ij})$$站点负载量 $r_i$ 反映了站点的繁忙程度。通过降低换乘点的平均负荷，查询系统可以引导乘客避开高峰期的人流密集区 。

### 核心约束条件体系

为了使搜索路径具有物理可行性，模型III必须满足以下严苛的约束条件 ：

1. 流量平衡约束（始末点连通性）：对于起点 $s$、终点 $e$ 及任意中间点 $i$，必须满足：$$\sum_{j} (x_{ij} + y_{ij}) - \sum_{j} (x_{ji} + y_{ji}) = \begin{cases} 1, & i = s \\ -1, & i = e \\ 0, & i \neq s, e \end{cases}$$这保证了路径是从起点出发、经中间站点连续跳转并最终抵达终点的完整有向链 。

2. 换乘次数上限约束：这是公交查询系统的最基本约束。用户通常无法接受超过3次的换乘。$$\sum_{(i,j) \in E} x_{ij} - 1 \le c$$
3. 单一路径互斥约束：在任意两相邻站点段 $(i, j)$ 之间，出行的逻辑不允许同时发生步行和乘车（除非是逻辑合并后的弧）。$$x_{ij} + y_{ij} \le 1$$
4. 0-1 整数约束：$$x_{ij}, y_{ij} \in \{0, 1\}$$
## 详细解答步骤：
1. 构建基础赋权邻接矩阵。
	- 首先调用数据读取程序，从 `1.1 公汽线路信息.txt` 和 `1.2 地铁线路信息.txt` 中提取原始站点序列。利用站点编号（S0001-S3957）作为矩阵索引，初始化乘车时间矩阵 $T^1$、费用矩阵 $P$ 和始发权重矩阵 $f$ 。
2. 量化步行时间并叠加弧集。
	- 将假设已知的站点间步行时间转化为步行邻接矩阵 $T^2$。在此步骤中，算法需要识别并剔除无效弧。例如，如果两站间的步行时间超过15分钟，则将其权值设为 $+\infty$，从而缩减搜索空间 。
3. 建立综合直达快表 $Q'$。
	- 将原有的数据库 $Q$ 扩展。如果站点 $i$ 与站点 $j$ 之间步行时间极短（如 $\le 5$ 分钟），则在数据库中将其标记为“步行直达”。这相当于在交通图中为每一个站点增加了数条通往邻近车站的“虚拟线路” 。
4. 确定最小换乘次数基准。
	- 利用矩阵运算确定起始点 $s$ 到 $e$ 的最少换乘次数 $c_{min}$。如果 $c_{min}=0$（即存在乘车或步行直达），则直接跳转至 **步骤8** 输出结果。否则，在 $c_{min}$ 的基础上扩展搜索 1 到 2 次换乘的可能路径 。
5. 执行广度优先邻接搜索。
	- 利用改进的 *Dijkstra* 算法，在换乘次数受限的前提下，搜索所有从 $s$ 到 $e$ 的可行路径。在此过程中，算法会记录每条路径经过的所有中间站点及其对应的交通模式，并将结果存入临时队列 $U$ 。
6. 计算多目标向量值。
	- 对队列 $U$ 中的每一个方案，代入前面的数学公式计算 $Z_1$ 到 $Z_4$ 的具体数值。特别地，对于涉及地铁换乘的方案，需额外加上地铁出入站所需的4分钟固定步行时间损耗 。
7. 应用分层序列排序逻辑。
	- 采用多目标分层序列排序法对方案集进行优劣排序。
		- **第一优先级**：换乘次数。
		- **第二优先级**：行程总时间 $Z_1$。
		- **第三优先级**：始发站比例 $Z_3$。
		-  **第四优先级**：站点负载 $Z_4$。 
	- 通过这种方式，算法不仅能给出物理上的最优路径，还能提供更具人性化的推荐，如“多走3分钟路，但可以上车有座且直达” 。
8. 输出综合建议方案集。
	- 将排名前三的方案输出到用户终端，方案应详细包含：每段路程的交通工具（如 L001、T1）、换乘站点、步行距离以及预估的到站时间。
## 关键参数 $\omega$ 与步行逻辑的敏感性分析
### 步行心理阈值 $\omega$ 的定义
该论文引入了 $\omega$ 作为用户对步行承受能力的量化指标。
1. **取值建议**：论文认为，在正常的交通场景下，普通乘客愿意为减少一次换乘而步行的最长时间约为10至15分钟。因此，在算法初始化阶段，任何大于15分钟的步行弧都被视为对目标函数极大的负贡献 。
2. **折算逻辑**：对于那些没有明确数据的站点对，论文通过站点编号对应的潜在地理坐标，按 $v = 1.2 \text{ m/s}$（平均步行速度）进行空间折算。这种方法虽然是一种近似，但在大规模公交线网模型中能有效解决数据缺失导致的问题 。
### 步行对换乘策略的影响
引入步行时间后，路径规划的逻辑发生了质变。复现发现，在考虑步行前，换乘只能发生在两线共有的站点上；而考虑步行后，换乘可以发生在物理距离接近的任意两个站点之间。这极大地增加了搜索图的边密度，但也为缩短出行时间提供了可能。

    def cal_min_transfer(self):
        """
        优化：用BFS计算最少换乘次数矩阵C
        """
        n = self.station_counter
        self.matrix_C = np.ones((n + 1, n + 1), dtype=int) * -1  # 初始化为-1（不可达）
        np.fill_diagonal(self.matrix_C, 0)  # 自身换乘次数为0

        print("\n 正在用BFS计算最少换乘矩阵...")
        # 构建邻接表
        adj = defaultdict(list)
        # 遍历直达数据库，构建“站点→可达站点”的邻接表（直达=1步，对应0次换乘）
        for s in self.direct_db:
            for e in self.direct_db[s]:
                adj[s].append(e)  # s→e有直达线路，邻接表添加边

        # 对每个节点做BFS，求到所有节点的最短路径（换乘次数=路径长度-1）
        for start in range(1, n + 1):
            # BFS初始化
            visited = {start: 0}  # key=节点，value=换乘次数
            queue = [start]
            
            while queue:
                current = queue.pop(0)  # 队列头出队
                current_trans = visited[current]
                
                # 遍历当前节点的所有可达节点（直达）
                for neighbor in adj.get(current, []):
                    # 未访问过 → 换乘次数=当前换乘次数+1
                    if neighbor not in visited:
                        visited[neighbor] = current_trans + 1
                        queue.append(neighbor)
            
            # 将BFS结果写入最少换乘矩阵
            for end in visited:
                self.matrix_C[start, end] = visited[end]

        print(f"  最少换乘矩阵计算完成！")
        print(f"   ├─ 矩阵维度：{self.matrix_C.shape}")
        print(f"   └─ 示例：站点1→10的最少换乘次数：{self.matrix_C[1, 10]}")

        """

        :param s_id: 起点数字ID
        :param e_id: 终点数字ID
        :return: res1（1次换乘方案）、res2（2次换乘方案）
        """
        res1, res2 = [], []
        n = self.station_counter

        # 直达判断
        if e_id in self.direct_db.get(s_id, {}):
            print(f"\n 起点{self.station_rev[s_id]}→终点{self.station_rev[e_id]}有直达方案，无需换乘！")
            return res1, res2
        
        min_trans = self.matrix_C[s_id, e_id]
        # 只搜索≤最少换乘次数的方案
        if min_trans == 1:
            # 只搜索1次换乘
            print("\n 正在搜索1次换乘方案...")
            for m in range(1, n + 1):
                if m == s_id or m == e_id:
                    continue
                # 检查s→m和m→e是否有直达方案
                if m not in self.direct_db.get(s_id, {}) or e_id not in self.direct_db.get(m, {}):
                    continue

                # 遍历所有直达组合
                for p1 in self.direct_db[s_id][m]:
                    l1, t1, f1, _, is1 = p1
                    for p2 in self.direct_db[m][e_id]:
                        l2, t2, f2, _, is2 = p2
                        # 总耗时 = 两段耗时 + 换乘时间
                        total_t = t1 + t2 + self.TRANS_BUS
                        # 总费用 = 两段费用之和
                        total_f = f1 + f2
                        # 始发站数
                        start_num = is1 + is2
                        # 站点负载（简化：出站数-入站数）
                        load = np.sum(self.matrix_A[m, :]) - np.sum(self.matrix_A[:, m])

                        res1.append([
                            s_id, m, e_id, l1, l2, total_t, total_f, start_num, load
                        ])
        elif min_trans == 2:
            # 搜索1、2次换乘
            print("\n 正在搜索1次换乘方案...")
            for m in range(1, n + 1):
                if m == s_id or m == e_id:
                    continue
                # 检查s→m和m→e是否有直达方案
                if m not in self.direct_db.get(s_id, {}) or e_id not in self.direct_db.get(m, {}):
                    continue

                # 遍历所有直达组合
                for p1 in self.direct_db[s_id][m]:
                    l1, t1, f1, _, is1 = p1
                    for p2 in self.direct_db[m][e_id]:
                        l2, t2, f2, _, is2 = p2
                        # 总耗时 = 两段耗时 + 换乘时间
                        total_t = t1 + t2 + self.TRANS_BUS
                        # 总费用 = 两段费用之和
                        total_f = f1 + f2
                        # 始发站数
                        start_num = is1 + is2
                        # 站点负载（简化：出站数-入站数）
                        load = np.sum(self.matrix_A[m, :]) - np.sum(self.matrix_A[:, m])

                        res1.append([
                            s_id, m, e_id, l1, l2, total_t, total_f, start_num, load
                        ])

        
            print(" 正在搜索2次换乘方案...")
            for m1 in range(1, n + 1):
                for m2 in range(1, n + 1):
                    if m1 == s_id or m1 == e_id or m2 == s_id or m2 == e_id or m1 == m2:
                        continue
                    # 检查s→m1、m1→m2、m2→e是否有直达方案
                    if (m1 not in self.direct_db.get(s_id, {}) or
                        m2 not in self.direct_db.get(m1, {}) or
                        e_id not in self.direct_db.get(m2, {})):
                        continue

                    # 遍历所有直达组合
                    for p1 in self.direct_db[s_id][m1]:
                        for p2 in self.direct_db[m1][m2]:
                            for p3 in self.direct_db[m2][e_id]:
                                l1, t1, f1, _, is1 = p1
                                l2, t2, f2, _, is2 = p2
                                l3, t3, f3, _, is3 = p3

                                total_t = t1 + t2 + t3 + 2 * self.TRANS_BUS
                                total_f = f1 + f2 + f3
                                start_num = is1 + is2 + is3
                                load = (np.sum(self.matrix_A[m1, :]) - np.sum(self.matrix_A[:, m1])) + \
                                    (np.sum(self.matrix_A[m2, :]) - np.sum(self.matrix_A[:, m2]))

                                res2.append([
                                    s_id, m1, m2, e_id, l1, l2, l3, total_t, total_f, start_num, load
                                ])
        
        # 转换为numpy数组便于后续排序
        res1 = np.array(res1) if res1 else np.array([])
        res2 = np.array(res2) if res2 else np.array([])

        print(f"  邻接算法搜索完成！")
        print(f"   ├─ 1次换乘方案数：{len(res1)}")
        print(f"   └─ 2次换乘方案数：{len(res2)}")
        return res1, res2

    def fang_sort(self, res, transfer_type):
        """
        分层序列法多目标排序
        优先级：换乘次数 → 总时间 → 始发站数（最大化） → 站点负载 → 费用
        :param res: 原始方案列表
        :param transfer_type: 1=1次换乘，2=2次换乘
        :return: 排序后的方案
        """
        if len(res) == 0:
            return np.array([])

        # 排序权重（高优先级对应大权重，始发数最大化用负号）
        weights = [1e8, 1e4, -1e2, 1, 1]  # 换乘次数→时间→始发数→负载→费用
        sort_keys = []

        for row in res:
            # 强制转换为数值类型
            if transfer_type == 1:
                # row结构：s, m, e, l1, l2, total_t, total_f, start_num, load
                t = float(row[5])       # 总耗时
                f = float(row[6])       # 总费用
                start_num = int(row[7]) # 始发站数
                load = float(row[8])    # 站点负载
                key = 1 * weights[0] + t * weights[1] + start_num * weights[2] + abs(load) * weights[3] + f * weights[4]
            else:
                # row结构：s, m1, m2, e, l1, l2, l3, total_t, total_f, start_num, load
                t = float(row[7])       # 总耗时
                f = float(row[8])       # 总费用
                start_num = int(row[9]) # 始发站数
                load = float(row[10])   # 站点负载
                key = 2 * weights[0] + t * weights[1] + start_num * weights[2] + abs(load) * weights[3] + f * weights[4]
            sort_keys.append(key)

        # 按排序键升序排列
        idx = np.argsort(sort_keys)
        sorted_res = res[idx]

        print(f" 分层序列排序完成！最优{transfer_type}次换乘方案已筛选")
        return sorted_res

    def model3(self, s_id, e_id):
        """
        模型Ⅲ：叠加步行弧的多模态查询（公汽+地铁+步行）
        核心：乘车弧x_ij + 步行弧y_ij，约束步行时间≤阈值
        :param s_id: 起点数字ID
        :param e_id: 终点数字ID
        :return: 多模态方案列表
        """
        res = []
        n = self.station_counter
        print(f"\n  模型Ⅲ搜索（步行阈值{self.WALK_THRESHOLD}分钟）...")

        # 遍历所有中间站，组合步行+乘车方案
        for m1 in range(1, n + 1):
            for m2 in range(1, n + 1):
                if m1 == s_id or m1 == e_id or m2 == s_id or m2 == e_id:
                    continue

                # 情况1：起点步行→m1 → 乘车m1→m2 → 乘车m2→终点
                if (self.walk_time[s_id, m1] <= self.WALK_THRESHOLD and
                    m1 in self.direct_db and m2 in self.direct_db.get(m1, {}) and
                    e_id in self.direct_db.get(m2, {})):
                    walk_t1 = self.walk_time[s_id, m1]
                    for p1 in self.direct_db[m1][m2]:
                        for p2 in self.direct_db[m2][e_id]:
                            l1, t1, f1, _, is1 = p1
                            l2, t2, f2, _, is2 = p2
                            total_t = walk_t1 + t1 + t2 + self.TRANS_BUS
                            total_f = f1 + f2
                            start_num = is1 + is2

                            res.append([
                                f"步行({walk_t1:.0f}min)", self.station_rev[m1], l1,
                                self.station_rev[m2], l2, self.station_rev[e_id],
                                total_t, total_f, start_num
                            ])

                # 情况2：起点乘车→m1 → 步行m1→m2 → 乘车m2→终点
                if (s_id in self.direct_db and m1 in self.direct_db.get(s_id, {}) and
                    self.walk_time[m1, m2] <= self.WALK_THRESHOLD and
                    e_id in self.direct_db.get(m2, {})):
                    walk_t2 = self.walk_time[m1, m2]
                    for p1 in self.direct_db[s_id][m1]:
                        for p2 in self.direct_db[m2][e_id]:
                            l1, t1, f1, _, is1 = p1
                            l2, t2, f2, _, is2 = p2
                            total_t = t1 + walk_t2 + t2 + self.TRANS_BUS
                            total_f = f1 + f2
                            start_num = is1 + is2

                            res.append([
                                self.station_rev[s_id], l1, f"步行({walk_t2:.0f}min)",
                                self.station_rev[m2], l2, self.station_rev[e_id],
                                total_t, total_f, start_num
                            ])

        # 按时间→费用→始发数排序
        if res:
            res = np.array(res)
            # 按时间升序、费用升序、始发数降序排序
            idx = np.lexsort((-res[:, 8].astype(int), res[:, 7].astype(float), res[:, 6].astype(float)))
            res = res[idx]

        print(f"  模型Ⅲ搜索完成！多模态方案数：{len(res)}")
        return res
