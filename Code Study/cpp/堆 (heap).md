# 堆（Heap）介绍

堆是**完全二叉树**为底层逻辑的经典数据结构，核心特性是**堆顶为最值**（大顶堆 / 小顶堆），且支持高效的**插入**和**删除堆顶**操作，时间复杂度均为**O(logn)**；同时堆也是实现**优先队列**的底层基础，在 C++ 中 STL 并未提供专门的堆容器，而是通过**算法函数**在序列容器（如`vector`）上实现堆操作，还封装了`priority_queue`（优先队列）作为堆的常用实用工具。

本文将从**堆的基础特性**（数据结构层面）、**C++ STL 堆操作**（核心实用）、**优先队列封装**、**经典适用场景**、**与 multiset 对比**几个维度讲解，贴合 C++ 实际开发，兼顾基础和使用。

## 一、堆的基础定义与核心特性

堆的本质是**满足堆序性的完全二叉树**，完全二叉树的结构保证了堆可以用**数组**高效存储（无需链式结构，节省空间），堆序性则决定了堆的核心价值 —— 快速获取最值。

### 1. 堆的两大类型

堆分为**大顶堆**和**小顶堆**，无特殊说明时默认指**大顶堆**，两者的堆序性规则如下：

- **大顶堆**：**每个父节点的值 ≥ 其左右子节点的值** → 堆顶（根节点）是整个堆的**最大值**。
- **小顶堆**：**每个父节点的值 ≤ 其左右子节点的值** → 堆顶（根节点）是整个堆的**最小值**。

**核心注意**：堆仅保证父节点与子节点的大小关系，**不保证左右子节点之间的顺序**，也不保证整个堆是严格有序的（这是和`set/multiset`的关键区别）。

### 2. 堆的核心特性总结

1. 结构：基于**完全二叉树**，适合数组存储，无冗余空间；
2. 最值：堆顶固定为全局最值（大 / 小顶堆），获取最值时间复杂度**O(1)**；
3. 操作：插入、删除堆顶为**O(logn)**，建堆为**O(n)**（远优于 O (nlogn)）；
4. 重复性：允许存储重复元素，无唯一性限制；
5. 灵活性：可快速切换大 / 小顶堆，适配不同最值需求。

## 二、堆的存储方式（数组实现）

完全二叉树的天然特性让堆可以用**一维数组**直接存储，无需指针，通过**索引计算**即可找到任意节点的父节点和左右子节点，这是堆高效实现的关键。

### 数组索引规则（从 0 开始，C++ 默认）

设数组中某个节点的索引为`i`（从 0 开始），则：

- 父节点索引：`(i - 1) / 2`（整数除法，向下取整）；
- 左子节点索引：`2 * i + 1`；
- 右子节点索引：`2 * i + 2`。

### 示例（大顶堆）

大顶堆`{9,7,8,5,6,3}`的完全二叉树结构与数组存储对应关系：

plaintext

```
        9(0)
       /   \
      7(1)  8(2)
     / \   /
    5(3)6(4)3(5)
```

数组存储：`[9,7,8,5,6,3]`，完全符合**父节点≥子节点**的大顶堆规则。

## 三、堆的核心基本操作

堆的操作围绕**维持堆序性**展开，核心操作有 3 个：**建堆**、**插入元素**、**删除堆顶**，所有操作的核心是 ** 上浮（sift up）**和**下沉（sift down）** 两个调整算法：

- **上浮**：插入元素时，将元素放在数组尾部，逐步与父节点比较并交换，直到恢复堆序性；
- **下沉**：删除堆顶 / 建堆时，将根节点与尾部元素交换（删除则弹出尾部），逐步与子节点比较并交换，直到恢复堆序性。

### 各操作步骤与时间复杂度

1. **建堆**：对非叶子节点从后往前依次执行下沉操作 → 时间复杂度**O(n)**（数学推导可得，非 O (nlogn)）；
2. **插入元素**：尾部追加 → 上浮调整 → 时间复杂度**O(logn)**（树的高度为 logn）；
3. **删除堆顶**：堆顶与尾部交换 → 弹出尾部 → 根节点下沉调整 → 时间复杂度**O(logn)**；
4. **获取最值**：直接取数组第一个元素（堆顶） → 时间复杂度**O(1)**。

## 四、C++ STL 中的堆操作（重点）

C++ STL**没有专门的堆容器**，而是在`<algorithm>`头文件中提供了**堆算法函数**，用于在**随机访问序列容器**（优先`vector`，也可用`deque`，不推荐`array`）上实现堆操作，**默认构建大顶堆**，支持自定义小顶堆。

### 1. 必备头文件

```
#include <algorithm> // 堆算法核心
#include <vector>    // 推荐的底层存储容器
```

### 2. STL 堆核心算法（6 个）

所有算法均接收**迭代器**作为参数，针对`[first, last)`区间操作，默认使用`<`运算符判断大小（大顶堆），自定义规则需传入比较函数。

|算法函数|功能说明|注意事项|
|---|---|---|
|`make_heap(first, last)`|对`[first, last)`区间建堆，**默认大顶堆**|建堆后容器成为堆结构，堆顶是`*first`|
|`push_heap(first, last)`|向已有的堆中**插入新元素**|需先将元素**追加到容器尾部**，再调用该函数（自动上浮调整）|
|`pop_heap(first, last)`|从堆中**删除堆顶元素**|仅将堆顶移到容器**尾部**，需手动调用`pop_back()`弹出（自动下沉调整）|
|`sort_heap(first, last)`|对堆进行**堆排序**，将堆转为**严格升序序列**|排序后**容器不再是堆结构**；默认大顶堆排序后为升序，小顶堆为降序|
|`is_heap(first, last)`|判断`[first, last)`区间**是否为堆**（默认判断大顶堆）|是返回`true`，否返回`false`|
|`is_heap_until(first, last)`|找到区间中**第一个不满足堆序性的元素迭代器**，默认判断大顶堆|可用于检查堆的有效范围|

### 3. 基础使用示例（大顶堆 + 小顶堆）

#### （1）默认大顶堆操作

```cpp
#include <algorithm>
#include <vector>
#include <iostream>
using namespace std;

int main() {
    vector<int> v{3,1,4,1,5,9,2,6};

    // 1. 建大顶堆
    make_heap(v.begin(), v.end());
    cout << "大顶堆顶：" << v[0] << endl; // 9（堆顶是第一个元素）

    // 2. 插入元素：先尾插，再push_heap
    v.push_back(8);
    push_heap(v.begin(), v.end());
    cout << "插入8后堆顶：" << v[0] << endl; // 9

    // 3. 删除堆顶：先pop_heap，再尾删
    pop_heap(v.begin(), v.end());
    v.pop_back();
    cout << "删除堆顶后堆顶：" << v[0] << endl; // 8

    // 4. 判断是否为堆
    if (is_heap(v.begin(), v.end())) {
        cout << "当前是堆结构" << endl; // 输出
    }

    // 5. 堆排序：转为升序序列
    sort_heap(v.begin(), v.end());
    cout << "堆排序后（升序）：";
    for (int num : v) cout << num << " "; // 1 1 2 3 4 5 6 8
    cout << endl;

    // 排序后不再是堆
    if (!is_heap(v.begin(), v.end())) {
        cout << "排序后不是堆结构" << endl; // 输出
    }

    return 0;
}
```

#### （2）自定义小顶堆操作

STL 堆默认用`<`构建大顶堆，构建**小顶堆**需传入**比较函数`greater<int>()`**（需包含`<functional>`头文件，部分编译器可省略），**所有堆算法都要统一传入比较函数**，否则会破坏堆序性。

```cpp
#include <algorithm>
#include <vector>
#include <iostream>
#include <functional> // greater<>
using namespace std;

int main() {
    vector<int> v{3,1,4,1,5,9,2,6};

    // 1. 建小顶堆，传入greater<int>()
    make_heap(v.begin(), v.end(), greater<int>());
    cout << "小顶堆顶：" << v[0] << endl; // 1

    // 2. 插入元素：尾插+push_heap，统一传比较函数
    v.push_back(0);
    push_heap(v.begin(), v.end(), greater<int>());
    cout << "插入0后堆顶：" << v[0] << endl; // 0

    // 3. 删除堆顶：pop_heap+尾删，统一传比较函数
    pop_heap(v.begin(), v.end(), greater<int>());
    v.pop_back();
    cout << "删除堆顶后堆顶：" << v[0] << endl; // 1

    // 4. 小顶堆的堆排序：转为降序序列
    sort_heap(v.begin(), v.end(), greater<int>());
    cout << "小顶堆排序后（降序）：";
    for (int num : v) cout << num << " "; // 9 6 5 4 3 2 1 1
    cout << endl;

    return 0;
}
```

## 五、C++ STL 优先队列`priority_queue`（堆的封装，推荐使用）

直接使用 STL 堆算法需要手动处理尾插 / 尾删，步骤繁琐，因此 C++ 在`<queue>`头文件中封装了**优先队列`priority_queue`**，其**底层就是堆**，提供了更简洁的堆操作接口，是实际开发中使用堆的**首选方式**。

### 1. 定义与模板参数

```
#include <queue> // 必备头文件
// 模板参数：<存储类型, 底层容器, 比较函数>
priority_queue<T, Container = vector<T>, Compare = less<T>> pq;
```

- **T**：存储的元素类型（如`int`、`string`）；
- **Container**：底层存储的序列容器，**默认`vector<T>`**，也可改为`deque<T>`；
- **Compare**：比较函数，**默认`less<T>`（大顶堆）**，改为`greater<T>`则为小顶堆。

### 2. 简化定义

- 大顶堆（默认）：`priority_queue<int> pq;`
- 小顶堆：`priority_queue<int, vector<int>, greater<int>> pq;`

### 3. `priority_queue`核心成员函数（简洁易用）

无需关心堆的调整细节，直接调用接口即可，所有操作均维持堆序性。

|成员函数|功能说明|时间复杂度|
|---|---|---|
|`push(val)`|插入元素 val 到堆中|O(logn)|
|`pop()`|删除堆顶元素|O(logn)|
|`top()`|获取堆顶元素（最值）|O(1)|
|`empty()`|判断优先队列是否为空|O(1)|
|`size()`|获取优先队列中元素的个数|O(1)|

### 4. `priority_queue`使用示例（大顶堆 + 小顶堆）

```cpp
#include <queue>
#include <iostream>
using namespace std;

int main() {
    // 1. 大顶堆（默认）
    priority_queue<int> pq1;
    // 插入元素
    pq1.push(3); pq1.push(1); pq1.push(5); pq1.push(1);
    cout << "大顶堆顶：" << pq1.top() << endl; // 5
    cout << "大顶堆大小：" << pq1.size() << endl; // 4
    // 删除堆顶
    pq1.pop();
    cout << "删除后堆顶：" << pq1.top() << endl; // 3

    // 2. 小顶堆（需指定模板参数）
    priority_queue<int, vector<int>, greater<int>> pq2;
    pq2.push(3); pq2.push(1); pq2.push(5); pq2.push(1);
    cout << "小顶堆顶：" << pq2.top() << endl; // 1
    pq2.pop();
    cout << "删除后堆顶：" << pq2.top() << endl; // 1

    // 遍历堆（无迭代器，需通过弹出堆顶实现）
    cout << "小顶堆遍历（从小到大）：";
    while (!pq2.empty()) {
        cout << pq2.top() << " ";
        pq2.pop();
    } // 1 3 5
    cout << endl;

    return 0;
}
```

### 5. 自定义类型的堆（如结构体）

若堆中存储自定义类型（如结构体），需**重载`<`运算符**（大顶堆）或**自定义比较函数**（小顶堆），示例如下（以存储学生成绩的大顶堆为例）：

```cpp
#include <queue>
#include <string>
#include <iostream>
using namespace std;

// 自定义结构体：学生成绩
struct Student {
    string name;
    int score;
    // 重载<运算符，定义大顶堆规则（分数高的优先）
    bool operator<(const Student& other) const {
        return this->score < other.score; // 大顶堆用<，小顶堆用>
    }
};

int main() {
    priority_queue<Student> pq;
    pq.push({"Tom", 85});
    pq.push({"Jerry", 95});
    pq.push({"Mike", 90});
    // 堆顶是分数最高的
    cout << "分数最高：" << pq.top().name << " " << pq.top().score << endl; // Jerry 95
    return 0;
}
```

## 六、堆的经典适用场景

堆的核心优势是**O (1) 获取最值、O (logn) 增删**，适合处理**需要频繁获取 / 更新最值**的场景，是算法和开发中的高频数据结构，经典场景如下：

1. **TopK 问题**（最常用）：找数组中**前 K 大 / 前 K 小**的元素，用小顶堆 / 大顶堆实现，时间复杂度**O(nlogK)**（远优于排序的 O (nlogn)）；
2. **优先队列实现**：任务调度、事件驱动、消息队列中，按优先级处理任务（如高优先级任务先执行）；
3. **堆排序**：原地排序算法，时间复杂度 O (nlogn)，空间复杂度 O (1)（但不稳定，实际开发中不如`sort`常用）；
4. **数据流的最值**：实时处理数据流，动态获取当前数据流的最大值 / 最小值（如股票实时价格的最高 / 最低值）；
5. **哈夫曼编码、迪杰斯特拉算法、Prim 算法**：这些经典算法的底层均使用堆优化效率。

### 经典示例：TopK 问题（找数组中前 K 大的元素，小顶堆实现）

思路：用**小顶堆**存储前 K 大的元素，堆顶是这 K 个元素中的最小值，遍历剩余元素时，若元素大于堆顶则替换堆顶并调整，最终堆中即为前 K 大的元素，时间复杂度 O (nlogK)。

```cpp
#include <queue>
#include <vector>
#include <iostream>
using namespace std;

// 找nums中前k大的元素，返回结果数组
vector<int> topK(vector<int>& nums, int k) {
    // 小顶堆（存储前k大的元素，堆顶是最小值）
    priority_queue<int, vector<int>, greater<int>> pq;
    for (int num : nums) {
        pq.push(num);
        // 堆大小超过k，弹出堆顶（最小的）
        if (pq.size() > k) {
            pq.pop();
        }
    }
    // 提取堆中元素
    vector<int> res;
    while (!pq.empty()) {
        res.push_back(pq.top());
        pq.pop();
    }
    return res;
}

int main() {
    vector<int> nums{3,1,4,1,5,9,2,6};
    int k = 3;
    vector<int> res = topK(nums, k);
    cout << "前" << k << "大的元素：";
    for (int num : res) cout << num << " "; // 5 6 9（小顶堆遍历为升序）
    return 0;
}
```

## 七、堆与`multiset`的核心对比

堆和`multiset`都是 C++ 中处理动态数据的高效结构，均支持 O (logn) 的增删，允许重复元素，但**设计目标和适用场景完全不同**，核心区别如下表（重点记**最值获取、有序遍历、使用场景**）：

|特性|堆（`priority_queue`）|`multiset`|
|---|---|---|
|底层实现|完全二叉树（数组存储）|红黑树（平衡二叉搜索树）|
|有序性|仅保证堆顶为最值，**整体无序**|全局**严格有序**（默认升序）|
|最值获取|仅能获取堆顶（单最值），O (1)|可获取首 / 尾（最小 / 最大），O (1)；支持任意区间最值|
|遍历方式|无迭代器，仅能通过弹出堆顶遍历|双向迭代器，支持任意有序遍历、区间访问|
|元素查找|无直接查找接口，需遍历，O (n)|支持`find/count/equal_range`，O(logn)|
|核心优势|极致高效的**最值获取与增删**|高效的**有序遍历、元素统计、区间查询**|
|适用场景|频繁获取 / 更新最值（TopK、优先队列）|有序存储、统计元素次数、区间查找|
|重复元素|允许，无特殊处理|允许，提供`count/equal_range`专门处理|

### 选择原则

- 若需要**频繁获取 / 删除最值**（如 TopK、任务调度）→ 选**堆（`priority_queue`）**，效率更高；
- 若需要**有序遍历、统计元素次数、查找任意元素**（如成绩排名、日志统计）→ 选 **`multiset`**；
- 若既需要最值，又需要有序遍历 → 优先考虑`multiset`（牺牲少量效率，换取灵活性）。

## 八、堆的核心总结

1. 堆是**完全二叉树**实现的高效数据结构，分大顶堆 / 小顶堆，堆顶为最值，允许重复元素；
2. 核心操作的时间复杂度：获取最值 O (1)，增删 O (logn)，建堆 O (n)；
3. C++ 中无专门堆容器，**首选`priority_queue`**（堆的封装，接口简洁），底层用`vector`+ 堆算法实现；
4. STL 堆算法（`make_heap/push_heap`等）适用于手动控制堆结构，需配合序列容器使用，默认大顶堆；
5. 堆的核心适用场景是**频繁获取 / 更新最值**（TopK、优先队列），与`multiset`的有序遍历形成互补；
6. 堆仅保证堆顶序，整体无序，无查找接口，这是与平衡二叉搜索树（红黑树）的本质区别。
