# 链式前向星 超详细全解

链式前向星的本质是**用静态数组模拟链表实现的邻接表**，是算法竞赛、高性能图处理中最主流的稀疏图存储结构。它完美解决了邻接矩阵的空间浪费、指针版邻接表的编码风险、vector 邻接表的常数开销问题，兼顾了空间效率、编码简洁性和运行速度。

下面从「底层原理→核心结构→逐步骤操作→完整示例→对比与误区→实战应用」全链路拆解，帮你彻底吃透。

---

## 一、先搞懂：为什么要有链式前向星？

在讲它之前，先回顾图的 3 种基础存储方式的痛点，你就能瞬间明白它的核心价值：

|存储方式|核心实现|空间复杂度|核心痛点|
|---|---|---|---|
|邻接矩阵|二维数组 `g[u][v]` 存 u 到 v 的边权|O(n²)|稀疏图（边数 m 远小于 n²）空间爆炸，比如 n=1e5 时，数组需要 1e10 个元素，直接内存溢出|
|指针版邻接表|每个顶点对应一个链表，链表节点存边信息|O(n+m)|指针操作易出错，有内存泄漏风险，编码繁琐，竞赛中极少用|
|vector 邻接表|数组套 vector：`vector<Edge> g[MAXN]`|O(n+m)|工程中常用，但动态扩容有均摊开销，极端场景下常数偏大，部分卡常题目会超时|

而**链式前向星**，就是用「两个固定大小的静态数组」完全模拟邻接表的链表结构，既保留了邻接表 O (n+m) 的优秀空间复杂度，又没有指针风险，常数极小，编码极简，是图论算法的 “万能存储底座”。

## 二、核心原理与结构拆解

链式前向星的核心逻辑：**给每个顶点维护一个单链表，链表的每个节点对应一条该顶点的出边；用静态数组代替指针实现链表，所有边集中存储在一个数组里**。

它只需要 2 个核心数组 + 1 个计数器，所有操作都围绕这三个元素展开。

### 1. 核心元素定义

#### （1）边集数组 `edges[]`

这是存储所有边的 “容器”，数组的每个元素对应一条边，是链表的节点。每条边包含 3 个核心属性（C++ 结构体实现）：

```cpp
struct Edge {
    int to;     // 【必填】这条边的终点（指向哪个顶点）
    int weight; // 【可选】边的权值，无权图可直接删除
    int next;   // 【核心】同起点的下一条边，在edges数组中的下标
};
```

- 关键理解：`next` 就是数组版的 “链表指针”，它存的不是内存地址，而是下一条边在`edges`数组里的索引。如果`next=-1`，说明这条边是当前顶点链表的最后一个节点，没有下一条出边了。

#### （2）头节点数组 `head[]`

这是每个顶点的 “链表头指针数组”，数组长度等于最大顶点数。

- `head[u]` 的含义：**顶点 u 的出边链表的第一个节点（第一条边），在 edges 数组中的下标**。
- 初始值：所有元素统一设为`-1`，代表该顶点暂时没有出边，链表为空。

#### （3）边计数器 `edgeCnt`

用来记录当前已经添加了多少条边，同时也是**下一条新边要存入 edges 数组的下标**。

- 初始值：`0`，第一条边存在`edges[0]`，第二条存在`edges[1]`，以此类推。
- 核心规则：每添加一条边，`edgeCnt` 自增 1。

## 三、核心操作：逐步骤拆解（最容易懵的部分，必看）

链式前向星只有 2 个核心操作：**初始化**、**添加边**，再加上配套的**遍历边**操作，就能完成所有图处理。

### 1. 初始化操作

初始化的核心是：把所有顶点的链表头设为空，边计数器归零。

```cpp
const int MAXN = 1e5 + 10; // 最大顶点数，根据题目调整
const int MAXM = 2e5 + 10; // 最大边数，无向图要开2倍！

Edge edges[MAXM];
int head[MAXN];
int edgeCnt;

// 初始化函数，每次建图前必须调用
void init() {
    memset(head, -1, sizeof(head)); // 所有head设为-1，链表为空
    edgeCnt = 0; // 边数归零
}
```

⚠️ 新手必踩坑：`head` 必须初始化为`-1`，不能是 0！因为`edges`数组的下标从 0 开始，初始化为 0 会把第 0 条边当成默认链表头，导致遍历完全错乱。

### 2. 添加边操作（头插法，核心中的核心）

链式前向星统一用**头插法**添加边，也就是每次把新边插到对应顶点链表的最前面，时间复杂度 O (1)，编码极简。

#### （1）有向图加边代码

```cpp
// 添加一条 从u到v、权值为w 的有向边
void addEdge(int u, int v, int w) {
    edges[edgeCnt].to = v;         // 1. 给新边赋值终点v
    edges[edgeCnt].weight = w;     // 2. 给新边赋值权值w
    edges[edgeCnt].next = head[u]; // 3. 新边的下一条边，指向u当前的链表头
    head[u] = edgeCnt++;            // 4. 更新u的链表头为新边，边计数器+1
}
```

#### （2）逐步骤拆解加边逻辑（配实例，一看就懂）

我们用一个极简的例子，一步步看加边时`head`和`edges`的变化，彻底搞懂头插法。

**示例场景**：给顶点 1 添加 2 条出边：先加`1→2（权3）`，再加`1→3（权5）`。

初始状态：`head[1] = -1`，`edgeCnt = 0`。

##### 步骤 1：添加第一条边 `1→2，权3`

1. 新边存在`edges[0]`，给`to`赋值为 2，`weight`赋值为 3；
2. `edges[0].next = head[1]` → 也就是`-1`，意思是这条边的下一条边为空；
3. `head[1] = 0`，把顶点 1 的链表头更新为新边的下标 0；
4. `edgeCnt` 自增为 1。

此时状态：

- `head[1] = 0`
- `edges[0] = {to=2, weight=3, next=-1}`

顶点 1 的链表：`head[1] → 0号边 → 结束`，只有一条出边`1→2`。

##### 步骤 2：添加第二条边 `1→3，权5`

1. 新边存在`edges[1]`，给`to`赋值为 3，`weight`赋值为 5；
2. `edges[1].next = head[1]` → 也就是`0`，意思是这条新边的下一条边，是之前的链表头（0 号边）；
3. `head[1] = 1`，把顶点 1 的链表头更新为新边的下标 1；
4. `edgeCnt` 自增为 2。

此时状态：

- `head[1] = 1`
- `edges[1] = {to=3, weight=5, next=0}`
- `edges[0] = {to=2, weight=3, next=-1}`

顶点 1 的链表变成：`head[1] → 1号边 → 0号边 → 结束`，现在有两条出边`1→3`、`1→2`。

#### （3）无向图加边

无向图的一条边，等价于**两条方向相反的有向边**。所以添加无向边`u-v`，只需要调用两次`addEdge`：

```cpp
// 添加无向边 u-v，权值w
addEdge(u, v, w);
addEdge(v, u, w);
```

⚠️ 新手必踩坑：无向图的`MAXM`必须开**2 倍最大边数**，因为每条边要存两次，否则会数组越界。

### 3. 遍历操作：遍历一个顶点的所有出边

遍历的本质，就是**从头节点开始，沿着 next 指针遍历整个单链表**，直到遇到`-1`结束。这是所有图论算法的核心遍历逻辑。

#### 遍历代码模板

```cpp
// 遍历顶点u的所有出边
for (int i = head[u]; i != -1; i = edges[i].next) {
    int v = edges[i].to;     // 拿到当前边的终点v
    int w = edges[i].weight; // 拿到当前边的权值w
    // 这里写对边 u->v 的处理逻辑
}
```

#### 对应上面的实例，遍历顶点 1 的出边：

1. 初始`i = head[1] = 1`，处理`edges[1]`：`1→3，权5`；
2. `i = edges[1].next = 0`，处理`edges[0]`：`1→2，权3`；
3. `i = edges[0].next = -1`，循环结束，遍历完成。

⚠️ 注意：因为用了头插法，**遍历顺序和边的添加顺序是相反的**。但这对 99% 的图论算法（DFS、BFS、最短路、拓扑排序等）完全没有影响，因为这些算法不关心边的遍历顺序。

## 四、完整可运行示例（建图 + 全遍历）

下面用一个 4 顶点 5 边的有向图，完整展示链式前向星的建图、遍历全过程，你可以直接复制运行。

```cpp
#include <iostream>
#include <cstring>
using namespace std;

// 最大顶点数和最大边数，根据题目调整
const int MAXN = 1005;
const int MAXM = 10005;

// 边的结构体
struct Edge {
    int to, weight, next;
} edges[MAXM];

// 核心数组和计数器
int head[MAXN], edgeCnt;

// 初始化
void init() {
    memset(head, -1, sizeof(head));
    edgeCnt = 0;
}

// 添加有向边
void addEdge(int u, int v, int w) {
    edges[edgeCnt].to = v;
    edges[edgeCnt].weight = w;
    edges[edgeCnt].next = head[u];
    head[u] = edgeCnt++;
}

int main() {
    init();
    // 图的信息：4个顶点，5条有向边
    int n = 4, m = 5;
    addEdge(1, 2, 3);  // 1→2，权3
    addEdge(1, 3, 5);  // 1→3，权5
    addEdge(2, 4, 2);  // 2→4，权2
    addEdge(3, 4, 1);  // 3→4，权1
    addEdge(4, 1, 4);  // 4→1，权4

    // 遍历每个顶点的所有出边
    for (int u = 1; u <= n; u++) {
        cout << "顶点" << u << "的出边：" << endl;
        for (int i = head[u]; i != -1; i = edges[i].next) {
            int v = edges[i].to;
            int w = edges[i].weight;
            cout << "  " << u << " → " << v << "，权值：" << w << endl;
        }
        cout << endl;
    }

    return 0;
}
```

#### 运行结果

```
顶点1的出边：
  1 → 3，权值：5
  1 → 2，权值：3

顶点2的出边：
  2 → 4，权值：2

顶点3的出边：
  3 → 4，权值：1

顶点4的出边：
  4 → 1，权值：4
```

## 五、核心优缺点与适用场景

### 优点

1. **空间效率极高**：仅存储实际存在的边，空间复杂度 O (n+m)，完美适配稀疏图，支持 1e5 级别的顶点和边；
2. **操作常数极小**：纯数组操作，无动态扩容、无指针寻址，运行速度远超 vector 邻接表，卡常题目首选；
3. **编码极简稳定**：固定模板，几乎不会写错，无内存泄漏风险，适配所有图论算法；
4. **适配特殊场景**：网络流等需要频繁访问反向边的场景，链式前向星有天然优势（`i`和`i^1`互为正反边）。

### 缺点

1. **遍历顺序与添加顺序相反**：头插法导致，不影响算法，但需要注意；
2. **不支持边的快速删除**：静态数组实现，删除边需要标记无效，无法直接释放空间，不适合频繁删边的场景；
3. **无法快速查找指定边**：要找 u→v 的边，必须遍历 u 的所有出边，时间复杂度 O (出度)。

### 适用场景

- 算法竞赛中的所有图论问题：最短路（Dijkstra/SPFA）、最小生成树、拓扑排序、强连通分量（Tarjan）、网络流、二分图匹配等；
- 大规模稀疏图处理：社交网络、路网、电路图等顶点多、边数少的场景。

## 六、新手必避的 8 个坑

1. **head 数组初始值错误**：必须初始化为`-1`，不能是 0，否则遍历逻辑完全错乱；
2. **边数数组开小**：无向图必须给`MAXM`开 2 倍最大边数，否则数组越界 RE；
3. **edgeCnt 初始值错误**：必须从 0 开始，否则反向边的`i^1`技巧失效；
4. **多组测试用例忘记重新初始化**：每组测试用例都要调用`init()`，否则上一组数据会污染当前组；
5. **顶点编号不统一**：顶点编号从 1 开始的话，head 数组要开`MAXN+1`，避免 0 号顶点越界；
6. **无权图保留 weight**：无权图可以直接删掉 weight 属性，节省空间和运行时间；
7. **遍历循环条件写错**：循环必须是`i != -1`，不能是`i > 0`，否则会漏掉 0 号边；
8. **无向图只加一条边**：无向边必须加正反两条，否则图会变成有向的，遍历不到反向边。

## 七、进阶技巧与实战应用

### 1. 网络流反向边技巧

网络流中，每条正向边都对应一条反向边，链式前向星的`edgeCnt`从 0 开始的特性，让反向边的访问变得极其简单：

- 正向边的下标是`i`，反向边的下标就是`i^1`（0 和 1 互为反向，2 和 3 互为反向，以此类推）。

示例：添加网络流的正反边

cpp

运行

```
// 正向边u→v，容量cap；反向边v→u，容量0
addEdge(u, v, cap);
addEdge(v, u, 0);

// 访问正向边i的反向边
int rev_edge = i ^ 1;
```

### 2. 链式前向星实现 BFS（无权图最短路）

下面是最常用的实战示例，用链式前向星实现 BFS，求起点到所有顶点的最短路径：

cpp

运行

```
#include <iostream>
#include <cstring>
#include <queue>
using namespace std;

const int MAXN = 1e5 + 10;
const int MAXM = 2e5 + 10;

struct Edge {
    int to, next;
} edges[MAXM];

int head[MAXN], edgeCnt;
int dist[MAXN]; // 存储起点到每个顶点的最短距离

void init() {
    memset(head, -1, sizeof(head));
    edgeCnt = 0;
    memset(dist, -1, sizeof(dist));
}

void addEdge(int u, int v) {
    edges[edgeCnt].to = v;
    edges[edgeCnt].next = head[u];
    head[u] = edgeCnt++;
}

// BFS求s到所有点的最短距离
void bfs(int s) {
    queue<int> q;
    dist[s] = 0;
    q.push(s);
    while (!q.empty()) {
        int u = q.front();
        q.pop();
        // 链式前向星核心遍历
        for (int i = head[u]; i != -1; i = edges[i].next) {
            int v = edges[i].to;
            if (dist[v] == -1) {
                dist[v] = dist[u] + 1;
                q.push(v);
            }
        }
    }
}

int main() {
    init();
    int n, m, s;
    cin >> n >> m >> s; // 输入顶点数、边数、起点
    for (int i = 0; i < m; i++) {
        int u, v;
        cin >> u >> v;
        addEdge(u, v);
        addEdge(v, u); // 无向图加双向边
    }
    bfs(s);
    for (int i = 1; i <= n; i++) {
        cout << "顶点" << s << "到顶点" << i << "的最短距离：" << dist[i] << endl;
    }
    return 0;
}
```

## 八、最终总结

链式前向星的核心，一句话就能概括：**用 head 数组存每个顶点的链表头，用 edges 数组存边节点，每个边的 next 指向下一条同起点的边，头插法加边，循环遍历链表**。

它是图论算法的 “基本功”，只要掌握了它，几乎所有图论问题的存储部分都能一键解决，剩下的只需要专注于算法逻辑本身。
