# 匈牙利算法详解

匈牙利算法是图论中解决**二分图匹配问题**的经典多项式时间组合优化算法，核心理论源自匈牙利数学家 Dénes Kőnig 的研究，1955 年由美国数学家 Harold Kuhn 系统化提出，分为两大核心应用分支：

1. **基础版**：解决**无权二分图的最大匹配问题**（最通用、最入门的版本，核心基于增广路定理）
2. **加权版**：解决**带权二分图的最小 / 最大权完美匹配问题**（又称 Kuhn-Munkres/KM 算法，解决经典指派问题）

---

## 一、必备基础概念

以二分图 G=(U, V, E) 为例，先明确核心术语，这是理解算法的前提：

- **二分图**：顶点集可拆分为两个互不相交的子集 U（左部）、V（右部），图中所有边的两个端点必分属 U 和 V，同一子集内没有任何边。
- **匹配**：一个边的集合，其中任意两条边都没有公共顶点（一个顶点最多只能属于一条匹配边）。
- **最大匹配**：二分图中边数最多的匹配（匹配规模最大化）。
- **完美匹配**：所有顶点都被覆盖的匹配（左部和右部的每个顶点都有对应的匹配边，是最大匹配的特例）。
- **交替路**：从一个未匹配顶点出发，依次经过「非匹配边→匹配边→非匹配边…」交替出现的路径。
- **增广路**：起点和终点都是未匹配顶点的交替路，是匈牙利算法的核心基石。
    
    - 核心性质：把增广路里的匹配边和非匹配边**翻转**（匹配变非匹配，非匹配变匹配），匹配的总边数会严格 + 1。
    - 核心定理（Berge 定理）：一个匹配是最大匹配，当且仅当图中不存在关于该匹配的增广路。

## 二、基础版：无权二分图最大匹配的匈牙利算法

### 1. 核心思想

枚举左部 U 的每一个顶点，通过深度优先搜索（DFS）或广度优先搜索（BFS）为其寻找增广路；每找到一条增广路，就翻转路径上的边的匹配状态，让总匹配数 + 1；当所有左部顶点处理完毕，找不到新的增广路时，就得到了二分图的最大匹配。

### 2. 算法步骤（DFS 实现，最常用）

以左部 U、右部 V 为例，步骤如下：

1. 初始化：定义`match[v]`数组，记录右部顶点 v 当前匹配的左部顶点，初始值为 - 1（未匹配）；最大匹配数`res=0`。
2. 依次遍历左部 U 中的每个顶点 u：
    
    a. 初始化`visited`数组，标记本轮搜索中已访问过的右部顶点，避免重复遍历和死循环。
    
    b. 调用 DFS 函数为 u 寻找增广路：
    
    i. 遍历 u 的所有邻接右部顶点 v。
    
    ii. 若 v 本轮未被访问，标记`visited[v]=true`。
    
    iii. 若 v 未被匹配（`match[v]==-1`），或者 v 当前匹配的左部顶点`match[v]`能找到其他可匹配的右部顶点（递归调用 DFS），则将`match[v]=u`，返回 true（找到增广路）。
    
    c. 若 DFS 返回 true，说明找到增广路，`res+=1`。
3. 遍历结束，`res`即为最大匹配数，`match`数组记录了最终的匹配关系。

### 3. 通俗示例

以极简的任务分配场景为例：

- 左部 U={工人 1, 工人 2, 工人 3}，右部 V={任务 A, 任务 B, 任务 C}
- 可胜任的边：工人 1-A、工人 1-B；工人 2-B、工人 2-C；工人 3-A
- 执行过程：
    
    1. 处理工人 1：匹配空闲的任务 A，匹配数 = 1。
    2. 处理工人 2：匹配空闲的任务 B，匹配数 = 2。
    3. 处理工人 3：目标任务 A 已被占用，递归检查工人 1 能否更换匹配；工人 1 的备选任务 B 已被占用，再递归检查工人 2 能否更换匹配；工人 2 的备选任务 C 空闲，成功找到增广路。
    4. 增广路翻转：工人 3-A、工人 1-B、工人 2-C，匹配数 = 3（完美匹配）。
    

### 4. 时间复杂度

- 基础 DFS 实现：最坏时间复杂度**O(VE)**，V 为顶点总数，E 为边总数。
- BFS 优化版（Hopcroft-Karp 算法）：批量寻找增广路，复杂度优化至**O(E√V)**，适合大规模二分图。

## 三、加权版：带权二分图完美匹配（KM 算法 / 指派问题匈牙利解法）

### 1. 问题背景

经典**指派问题**：n 个工人分配 n 个任务，工人 i 做任务 j 的成本 / 收益为 w [i][j]，要求每个工人恰好做一个任务、每个任务恰好由一个工人完成，求总成本最小 / 总收益最大的分配方案，即二分图的**最小 / 最大权完美匹配**。

### 2. 核心思想

通过给每个顶点设置**可行顶标**（左部顶标`u[i]`，右部顶标`v[j]`），满足约束：`u[i] + v[j] ≥ w[i][j]`（最小权场景）或`u[i] + v[j] ≤ w[i][j]`（最大权场景）；仅保留满足`u[i] + v[j] = w[i][j]`的边，构建**相等子图**。

算法核心是不断调整顶标，扩大相等子图的规模，直到在相等子图中找到完美匹配。根据 Kuhn-Munkres 定理，此时的完美匹配就是原问题的最优权完美匹配。

### 3. 核心步骤（最小权指派问题，矩阵版）

1. 矩阵规约：每行减去该行的最小值，每列减去该列的最小值，使矩阵每行每列都出现 0 元素。
2. 寻找独立零元素：尝试找到 n 个位于不同行、不同列的 0 元素（独立零）。若找到，这些位置就是最优分配方案，算法结束。
3. 覆盖零元素：用最少的直线覆盖矩阵中所有的 0 元素。
4. 矩阵调整：未被直线覆盖的元素减去其中的最小值，直线交叉处的元素加上该最小值，回到步骤 2 继续迭代。

### 4. 时间复杂度

标准 KM 算法实现的最坏时间复杂度为**O(n³)**，n 为二分图单侧的顶点数。

## 四、典型应用场景

1. 基础无权版：任务分配、座位安排、二分图判定、网络流简化问题、图像特征点匹配、婚恋配对场景。
2. 加权指派版：人员排班、物流调度、资源最优分配、路径规划、航班机位分配、广告投放最优匹配。

---

## 五、补充说明

- 匈牙利算法是多项式时间算法，彻底避免了暴力枚举匹配方案的指数级复杂度，是组合优化中的经典算法。
- KM 算法是匈牙利算法的加权推广，继承了增广路的核心逻辑，同时通过顶标系统实现了权重的最优解。
- 若二分图不存在完美匹配，KM 算法可通过补全虚拟顶点、设置权重为 0 / 无穷大，转化为最大匹配下的最优权解。
