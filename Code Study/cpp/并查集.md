# 并查集（DSU/Union-Find）原理详解

并查集（Disjoint Set Union，简称 DSU，也叫并查集 / 联合查找集），是一种**专门处理不相交集合的动态合并与查询**的树形数据结构，核心解决「动态连通性」问题：**快速判断两个元素是否属于同一集合、快速合并两个集合**，是算法竞赛、图论、工程开发中极其常用的轻量数据结构。

---

## 一、核心设计思想

并查集的本质是**一个森林（多棵互不相交的有根树）**，核心规则极简：

1. 一个集合对应一棵树，**树的根节点是该集合的唯一标识（代表元）**；
2. 每个节点仅需记录自己的父节点，无需存储完整的树结构；
3. 两个元素属于同一集合，当且仅当它们的根节点完全相同；
4. 合并两个集合，只需把其中一棵树的根节点，挂载到另一棵树的根节点下。

它的核心优势是：经过优化后，单次合并 / 查询操作的均摊时间复杂度**近似常数级 O (1)**，远超数组、链表等其他数据结构处理同类问题的效率。

---

## 二、三大基础操作（朴素版）

我们用数组`parent[]`存储每个节点的父节点，元素编号映射为连续整数（非整数元素可通过哈希映射为整数），先看最基础的实现逻辑。

### 1. 初始化 MakeSet

**功能**：给每个元素创建一个独立的集合，自己就是集合的根。

**实现规则**：对每个元素`i`，设置`parent[i] = i`，即父节点指向自己。

示例：初始化 5 个元素，`parent = [0,1,2,3,4]`，每个元素自成一个集合。

### 2. 查询 Find

**功能**：找到元素`x`所在集合的根节点（代表元），这是判断连通性的核心。

**朴素实现逻辑**：递归 / 迭代向上查找父节点，直到找到「父节点等于自身」的节点，即为根。

```cpp
// 朴素版Find（无优化）
int find(int x) {
    if (parent[x] != x) {
        return find(parent[x]); // 向上找父节点
    }
    return x; // 找到根节点
}
```

示例：若`parent[1]=2`，`parent[2]=3`，`parent[3]=3`，则`find(1)`会逐级向上，最终返回根节点 3。

### 3. 合并 Union

**功能**：将元素`x`和`y`所在的两个集合，合并为一个集合。

**朴素实现逻辑**：

1. 分别找到`x`的根`rootX`、`y`的根`rootY`；
2. 若`rootX == rootY`，说明两个元素已在同一集合，无需合并；
3. 若不同，将其中一个根的父节点指向另一个根，完成合并。

```cpp
// 朴素版Union（无优化）
void unionSet(int x, int y) {
    int rootX = find(x);
    int rootY = find(y);
    if (rootX != rootY) {
        parent[rootY] = rootX; // 把rootY的树挂载到rootX下
    }
}
```

## 三、朴素版的核心缺陷

朴素实现存在致命问题：**合并时无控制，树会退化成一条链表**，导致`find`操作的时间复杂度退化为 O (n)，完全失去效率优势。

举个极端例子：依次执行`union(1,2)`、`union(2,3)`、`union(3,4)`、...、`union(n-1,n)`，最终树会变成一条长链`1→2→3→...→n`，此时`find(1)`需要遍历整个链表，效率极低。

为了解决这个问题，并查集有两个核心优化，二者结合才能发挥出极致性能。

---

## 四、两大核心优化（标准并查集必备）

### 优化 1：路径压缩（Path Compression）

**作用对象**：`find`查询操作

**核心思路**：在查找根节点的过程中，**把沿途所有节点的父节点，直接指向最终的根节点**，彻底拉平树的高度，让后续查询一步到位。

这是一个「一劳永逸」的优化：一次查询的开销，会让后续所有相关节点的查询都变成 O (1)。

```cpp
// 带路径压缩的Find（递归版，简洁常用）
int find(int x) {
    if (parent[x] != x) {
        parent[x] = find(parent[x]); // 核心：路径压缩，把当前节点的父直接设为根
    }
    return parent[x];
}

// 迭代版（避免递归栈溢出，数据量极大时推荐）
int find(int x) {
    int root = x;
    // 先找到根节点
    while (parent[root] != root) {
        root = parent[root];
    }
    // 路径压缩：把沿途所有节点的父直接指向根
    while (parent[x] != root) {
        int next = parent[x];
        parent[x] = root;
        x = next;
    }
    return root;
}
```

**示例**：原链为`1→2→3→4`（根是 4），执行`find(1)`时，不仅返回 4，还会把 1、2 的父节点直接设为 4。后续再查询 1、2，直接一步找到根。

---

### 优化 2：按秩合并（Union by Rank/Size）

**作用对象**：`union`合并操作

**核心思路**：合并两个集合时，**永远把更小 / 更矮的树，挂载到更大 / 更高的树的根节点下**，避免树的高度无限制增长，始终保持树的平衡。

这里的「秩」有两种常用实现，效果几乎一致：

#### 方案 1：按集合大小（Size）合并

用`size[]`数组记录每个根节点对应集合的节点总数，合并时把小集合合并到大集合下，只有大集合的大小会更新。

#### 方案 2：按树的高度（Rank）合并

用`rank[]`数组记录每个根节点对应树的高度，合并时把矮树合并到高树下；只有当两棵树高度完全相同时，合并后新根的秩才 + 1。

```cpp
// 带按大小合并的完整实现（算法竞赛标准写法）
const int MAXN = 1e5 + 10;
int parent[MAXN];
int size_[MAXN]; // 记录每个集合的大小，仅根节点的size有效

// 初始化
void init(int n) {
    for (int i = 0; i < n; i++) {
        parent[i] = i;
        size_[i] = 1; // 每个集合初始大小为1
    }
}

// 带路径压缩的find
int find(int x) {
    return parent[x] == x ? x : (parent[x] = find(parent[x]));
}

// 带按大小合并的union
void unionSet(int x, int y) {
    int rootX = find(x);
    int rootY = find(y);
    if (rootX == rootY) return;
    // 小集合合并到大集合下
    if (size_[rootX] < size_[rootY]) swap(rootX, rootY);
    parent[rootY] = rootX;
    size_[rootX] += size_[rootY]; // 更新大集合的大小
}
```

## 五、时间复杂度

同时使用**路径压缩 + 按秩合并**的并查集，单次`find`/`union`操作的**均摊时间复杂度为 O (α(n))**，其中 α 是阿克曼函数的反函数。

阿克曼函数是一个增长极快的函数，对应的反函数 α(n) 增长极其缓慢：对于$n≤10^{600}$（远大于宇宙中原子的总数），α(n) 的值都不会超过 5。因此在实际工程和算法竞赛中，我们可以认为并查集的单次操作是 ** 常数级 O (1)** 的效率。

## 六、核心特性与典型应用

### 核心特性

- 优势：合并、查询效率极高，代码极简，内存占用低，适合处理大规模动态连通性问题；
- 短板：**不支持高效的集合拆分操作**，仅适合「先拆分、再合并、无拆分」的场景。

### 典型应用场景

1. **连通性判断**：无向图中两个节点是否连通、网络设备是否互通、朋友圈是否认识；
2. **最小生成树**：Kruskal 算法的核心，用并查集判断边的两个端点是否已连通，避免生成环；
3. **环检测**：快速判断无向图中是否存在环（合并时若两个节点已在同一集合，说明加边会形成环）；
4. **经典算法题**：省份数量（LeetCode 547）、冗余连接（LeetCode 684）、岛屿数量、等式方程的可满足性等。
